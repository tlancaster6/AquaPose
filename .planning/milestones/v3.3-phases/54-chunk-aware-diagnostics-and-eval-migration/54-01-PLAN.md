<objective>
Restructure diagnostic cache from per-stage flat files to per-chunk single-cache layout.

Purpose: Enable diagnostic caching in multi-chunk mode. The old layout (one cache per stage in a flat diagnostics/ dir) cannot support multi-chunk runs. The new layout stores one `cache.pkl` per chunk containing the full PipelineContext after reconstruction, plus a `manifest.json` summarizing all chunks.

Output: DiagnosticObserver writes `diagnostics/chunk_NNN/cache.pkl`; orchestrator removes diagnostic+chunk mutual exclusion; manifest.json written at run completion.
</objective>

  <action>
Rewrite DiagnosticObserver:

1. Add `chunk_idx: int = 0` constructor parameter. This determines the subdirectory name.

2. Change `_write_stage_cache` to no longer write per-stage files. Instead, accumulate the context reference on each StageComplete (keep `self.stages` dict for in-memory Jupyter use as before).

3. Add `_write_chunk_cache` called from `_on_pipeline_complete`. This:
   - Creates `diagnostics/chunk_{self._chunk_idx:03d}/` directory
   - Writes `cache.pkl` containing an envelope dict: `{run_id, timestamp, version_fingerprint, context}` where `context` is the final PipelineContext (from the last StageComplete event)
   - Uses atomic write (tempfile + os.replace) per existing pattern

4. Add `_write_manifest` method. Called from `_on_pipeline_complete`. Writes/updates `diagnostics/manifest.json` with:
   - `run_id`: from PipelineStart event
   - `total_frames`: from context.frame_count (may be chunk-local — orchestrator will set the real total later)
   - `chunk_size`: from context or None
   - `version_fingerprint`: from context_fingerprint()
   - `chunks`: list of dicts, each with `{index, start_frame, end_frame, stages_cached}`

   The manifest is APPENDED to (read existing, add this chunk's entry, write back). For the first chunk it creates fresh. Use `stages_cached` = list of stage names in `self.stages.keys()`.

5. In `_on_pipeline_complete`, store the final context from the last captured StageComplete so it can be written as the single cache.

6. Update `load_stage_cache` in core/context.py: Add a new function `load_chunk_cache(path)` that loads from the new chunk cache layout (same envelope format, just different path expectations). Keep `load_stage_cache` working for backward compat but have it also accept the new format. On fingerprint mismatch: log a warning but return the context anyway (no StaleCacheError for fingerprint issues — per user decision).

Important: The per-stage cache files (`detection_cache.pkl`, `tracking_cache.pkl`, etc.) are NO LONGER written. Only the single `cache.pkl` per chunk. The StageSnapshot in-memory dict is kept for Jupyter.
