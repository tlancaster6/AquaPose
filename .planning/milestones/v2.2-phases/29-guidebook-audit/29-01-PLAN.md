# Plan 29-01: Audit and Fix Stale Content

## Goal

Remove all stale references and update GUIDEBOOK.md to accurately reflect the shipped v2.1 codebase. After this plan, every section of the guidebook matches what is actually in the code.

## Context

The current GUIDEBOOK.md has several inaccuracies relative to the shipped v2.1 codebase:

1. **Source Layout (Section 4)** shows a clean `core/` hierarchy but the actual codebase has both `core/` sub-packages AND legacy top-level packages (`calibration/`, `reconstruction/`, `segmentation/`, `tracking/`, `visualization/`, `synthetic/`, `io/`). The layout tree must reflect reality.
2. **Milestone History (Section 15)** lists v2.1 as "in progress" — it shipped 2026-02-28.
3. **Section 16 (Definition of Done)** must be deleted — per user decision, roadmap has per-phase success criteria; guidebook shouldn't duplicate.
4. **Section 18 (Discretionary Items)** must be deleted — guidebook isn't the right place for it.
5. **Observer list (Section 10)** says "Alpha observers: timing, raw export (HDF5), visualization, diagnostics" but the actual observers are: `animation_observer`, `console_observer`, `diagnostic_observer`, `hdf5_observer`, `overlay_observer`, `tracklet_trail_observer`, plus `observer_factory` and `timing`. This list must be accurate.
6. **CLI (Section 14)** needs verification — check if `aquapose run` is still the only subcommand or if there are others.
