  <action>
Create `scripts/tune_threshold.py` with the following behavior:

**CLI interface** (argparse):
- `fixture_path` (positional): Path to midline fixture NPZ file
- `--n-frames` (default 15): Number of frames to evaluate
- `--min-threshold` (default 10): Lower bound of grid search
- `--max-threshold` (default 100): Upper bound of grid search
- `--step` (default 5): Step size for grid search
- `--top-n` (default 3): Number of top candidates for full Tier 2 evaluation

**Grid search phase** (Tier 1 only, for speed):
1. Generate threshold candidates: range(min_threshold, max_threshold + step, step) → list of floats
2. For each threshold, call `run_evaluation(fixture_path, n_frames=n_frames, backend="dlt", outlier_threshold=threshold)`
3. Extract from results: `tier1.overall_mean_px`, `tier1.fish_reconstructed`, `tier1.fish_available`
4. Compute a composite score for ranking. The scoring formula should balance error and yield:
   - `yield_ratio = fish_reconstructed / fish_available` (1.0 = perfect yield)
   - `target_yield = 9 / (fish_available / frames_evaluated)` — normalized target based on 9 fish per frame
   - Penalize low yield heavily: `score = overall_mean_px / max(yield_ratio, 0.01)` (lower is better). This means a threshold that halves fish count effectively doubles its "score" (worse).
   - Alternatively, simpler: `score = overall_mean_px * (1.0 + max(0, 1.0 - yield_ratio) * 10.0)` — penalizing yield drops by 10x the deficit. Use whichever is cleaner. The key insight from CONTEXT.md: "A low reprojection error is useless if we get there by throwing away all of our data."
5. Print a console table with columns: `Threshold | Mean Error (px) | Fish Reconstructed | Fish Available | Yield % | Score`
6. Sort by score (ascending = best first)

**Full evaluation phase** (Tier 1 + Tier 2 on top N):
1. Take top N candidates by score
2. Run baseline: `run_evaluation(fixture_path, n_frames=n_frames, backend="triangulation")`
3. For each top candidate, `run_evaluation(fixture_path, n_frames=n_frames, backend="dlt", outlier_threshold=threshold)` (already computed in grid search — but now we need Tier 2 which is included in run_evaluation output)
4. Note: run_evaluation already computes Tier 2, so the grid search results from phase 1 already include it. Optimization: just re-use the EvalResults from the grid search for top N candidates rather than re-running. Only run the baseline once.

**Comparison report** (side-by-side):
For each top candidate threshold, print:
```
=== Comparison: DLT (threshold=XX) vs Baseline (triangulation) ===

Overall:
  DLT:      mean=X.XX px, max=X.XX px, yield=N/M (XX%)
  Baseline: mean=X.XX px, max=X.XX px, yield=N/M (XX%)

Per-Fish Breakdown:
  Fish | DLT Mean (px) | DLT Status | Baseline Mean (px) | Baseline Status
  ---- | -------------- | ---------- | ------------------- | ---------------
  0    |     12.34      |     OK     |       15.67         |       OK
  1    |      N/A       |   MISSING  |        8.90         |       OK
  ...

Tier 2 Stability (max displacement in mm):
  Fish | DLT Max Disp | Baseline Max Disp
  ---- | ------------ | -----------------
  ...
```

Where "Status" is OK (reconstructed), MISSING (not in results), or LOW_CONF (is_low_confidence=True from Midline3D — but since we only have Tier1Result which doesn't carry is_low_confidence, use OK/MISSING only).

For the per-fish breakdown: iterate all fish IDs present in either DLT or baseline results. Use `tier1.per_fish` for mean_px values. Fish present in per_fish = reconstructed (OK), absent = MISSING.

For Tier 2 stability per fish: get max displacement across all dropout cameras for each fish from `tier2.per_fish_dropout`. Print the per-fish max as a single value.

**Important implementation notes:**
- Use a temporary directory for output_dir during grid search runs (via tempfile.TemporaryDirectory) to avoid polluting the fixture directory with eval_results.json from each threshold
- No output files are committed — everything is printed to console (per CONTEXT.md: "Grid search output is ephemeral")
- Follow the measure_baseline.py pattern for argparse and main() structure
- Print a final recommendation line: "Recommended threshold: XX.X px (score: Y.YY)" for the top-1 candidate
