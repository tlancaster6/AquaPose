**Midline3D (reconstruction.py):**

Make `control_points`, `knots`, `degree`, and `arc_length` optional to support raw-keypoint representation:

1. Change `control_points: np.ndarray` to `control_points: np.ndarray | None = None` — None when spline fitting was not performed.
2. Change `knots: np.ndarray` to `knots: np.ndarray | None = None` — None when no spline.
3. Change `degree: int` to `degree: int | None = None` — None when no spline.
4. Change `arc_length: float` to `arc_length: float | None = None` — None when no spline.
5. Add `points: np.ndarray | None = None` field (after `frame_index`) — raw 3D keypoints, shape `(N, 3)`, float32. This holds the primary reconstruction output when spline is disabled. When spline IS enabled, this holds the same data as `triangulated_points` (both populated for backward compat).
6. Add docstring explaining: `points` holds raw triangulated 3D keypoints, `control_points` holds B-spline control points when spline fitting was performed. At least one must be non-None.
7. Update the class docstring to describe both representations (raw-keypoint and spline-fitted).
8. Keep `triangulated_points` as-is (it was already optional, used for diagnostic data).

Note on field ordering: `points` must be placed after existing required fields but before the optional ones. Since we're making `control_points`, `knots`, `degree`, `arc_length` optional with defaults, the only remaining required fields are `fish_id`, `frame_index`, `half_widths`, `n_cameras`, `mean_residual`, `max_residual`. Reorder the fields so required fields come first, then all optional fields. The new field order should be:

```python
# Required fields
fish_id: int

half_widths: np.ndarray
n_cameras: int
mean_residual: float
max_residual: float

points: np.ndarray | None = None
control_points: np.ndarray | None = None
knots: np.ndarray | None = None
degree: int | None = None
arc_length: float | None = None
is_low_confidence: bool = False
per_camera_residuals: dict[str, float] | None = None
centroid_z: float | None = None
z_offsets: np.ndarray | None = None
triangulated_points: np.ndarray | None = None
per_point_inlier_cameras: list[list[str]] | None = None
```

**DltBackend (dlt.py):**

2. In `_reconstruct_fish()`, after building `pts_3d_arr` and doing z-flattening, add a branch:

   **When `self._spline_enabled is False` (new default path):**
   - Skip `fit_spline()` call entirely.
   - Skip spline residual computation (the `scipy.interpolate.BSpline` block).
   - Build `Midline3D` with:
     - `points=pts_3d_arr_full` — a full `(n_body_points, 3)` array where valid points are filled from triangulation and invalid points are NaN. This is similar to the existing `tri_pts` construction.
     - `control_points=None`, `knots=None`, `degree=None`, `arc_length=None`
     - `half_widths`, `n_cameras`, `mean_residual`, `max_residual`, etc. computed from the raw triangulation data (per-point residuals from `tri_result.mean_residuals`, not spline residuals).
     - Use the mean of `per_point_residuals` for `mean_residual` and max for `max_residual` (these are the raw triangulation residuals, not spline reprojection residuals).
   - Remove the `_MAX_ENDPOINT_GAP` check (only relevant for spline extrapolation) — move it inside the spline-enabled branch.
   - Remove the `self._min_body_points` check (spline minimum) — in raw mode, any fish with >= 2 valid points should be kept. Add a `_MIN_RAW_BODY_POINTS = 2` constant.

   **When `self._spline_enabled is True` (backward compat path):**
   - Keep existing behavior exactly: endpoint gap check, spline fitting, spline residual computation, control_points in Midline3D.
   - Also populate `points` field with the same data as `triangulated_points` for consistency.

3. Update the `__init__` to only precompute `_spline_knots` and `_min_body_points` when `spline_enabled=True`. When False, set `_min_body_points = _MIN_RAW_BODY_POINTS`.
