    **1. Create `src/aquapose/training/yolo_obb.py`:**
    Thin wrapper around ultralytics YOLO OBB training. Function `train_yolo_obb(data_dir: Path, output_dir: Path, *, epochs: int = 100, batch_size: int = 16, device: str | None = None, val_split: float = 0.2, imgsz: int = 640, model_size: str = "s") -> Path`:

    - Wraps `ultralytics.YOLO("yolov8{model_size}-obb.pt").train(data=..., epochs=..., batch=..., device=..., project=..., name=...)`.
    - `data_dir` should contain a YOLO-format dataset with `data.yaml`. The function looks for `data.yaml` in data_dir and passes it to ultralytics.
    - Auto-detects device if None (cuda if available, else cpu).
    - After training, copies best.pt from ultralytics output to `output_dir/best_model.pt` and last.pt to `output_dir/last_model.pt` for consistent naming.
    - Uses MetricsLogger from common.py to log a summary line after training completes (ultralytics handles its own epoch logging internally, so we just log the final results).
    - Returns path to best model weights.

    **2. Create `src/aquapose/training/pose.py`:**
    Keypoint/pose regression training. Function `train_pose(data_dir: Path, output_dir: Path, *, epochs: int = 100, batch_size: int = 8, lr: float = 1e-4, val_split: float = 0.2, patience: int = 20, num_workers: int = 4, device: str | None = None, backbone_weights: Path | None = None, unfreeze: bool = False, n_keypoints: int = 6) -> Path`:

    - Builds a pose regression model: _UNet encoder (from segmentation.model) with the decoder replaced by a regression head that outputs (B, n_keypoints*2) — flattened (x,y) coordinates normalized to [0,1] in crop space.
    - The regression head: takes the encoder's final feature map, applies AdaptiveAvgPool2d(1), flatten, Linear(enc_features, 256), ReLU, Linear(256, n_keypoints*2), Sigmoid.
    - If `--backbone-weights` provided: loads state_dict from the file, extracts encoder weights (keys starting with "enc"), loads them into the model's encoder, and freezes all encoder parameters (requires_grad=False).
    - If `--unfreeze` provided (only meaningful with --backbone-weights): does NOT freeze encoder params, enabling end-to-end fine-tuning. Without --unfreeze, encoder is frozen for transfer learning.
    - If no --backbone-weights: trains from ImageNet-pretrained MobileNetV3-Small encoder (unfrozen).
    - Loss: MSE between predicted and ground-truth keypoint coordinates.
    - Dataset: expects data_dir with COCO-format keypoint annotations. Create a `KeypointDataset` class in this file (or in datasets.py) that loads COCO JSON with keypoint annotations and returns (image_tensor [3,128,128], keypoints_tensor [n_keypoints*2]). Keypoints normalized to [0,1] crop coordinates.
    - Uses EarlyStopping (mode="min" for keypoint error), MetricsLogger (fields: train_loss, val_error), save_best_and_last.
    - Best metric = val keypoint error (lower is better, mean Euclidean distance in normalized coords).
    - Differential LR when backbone is unfrozen: encoder lr * 0.1, head lr * 1.0.
    - Returns path to best model.
