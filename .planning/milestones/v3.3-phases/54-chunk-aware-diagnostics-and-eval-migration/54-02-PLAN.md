Migrate EvalRunner and TuningOrchestrator to load from the new per-chunk cache layout.

Purpose: The eval and tuning tools currently expect per-stage cache files (`detection_cache.pkl`, etc.) in a flat `diagnostics/` directory. With Plan 01's restructuring, caches are now per-chunk (`diagnostics/chunk_NNN/cache.pkl`). This plan updates the evaluation suite to discover, load, and merge chunk caches.

Output: EvalRunner and TuningOrchestrator work with both single-chunk and multi-chunk diagnostic runs.
</objective>

1. Rewrite `_discover_caches()` in EvalRunner:
   - Read `diagnostics/manifest.json` if it exists to get chunk list and metadata
   - If no manifest, glob for `diagnostics/chunk_*/cache.pkl` and sort by directory name
   - For each chunk, call `load_chunk_cache()` (from Plan 01) to get the PipelineContext
   - Return a merged/combined view

2. Add `_merge_chunk_contexts()` method:
   - Takes list of (chunk_idx, start_frame, PipelineContext) tuples
   - Merges per-frame lists (detections, annotated_detections, midlines_3d) by concatenation with frame offset
   - Merges per-camera dicts (tracks_2d) by extending tracklet lists (adjusting frame indices in tracklets by chunk start_frame)
   - Merges tracklet_groups by concatenation
   - Returns a single synthetic PipelineContext with total frame_count = sum of all chunk frame_counts
   - camera_ids taken from first chunk (should be consistent)

3. Update `run()` to use the merged context instead of per-stage contexts. Since we now have ONE cache per chunk (not per-stage), the evaluation logic simplifies: load all chunks, merge into one big context, then evaluate each stage from that merged context.

4. The `_STAGE_KEYS` concept changes: instead of probing for per-stage cache files, we check which fields are populated on the merged context (e.g., `ctx.detections is not None` means detection stage was cached).

5. For single-chunk runs, this is trivially a one-element merge — behavior should be identical to the old per-stage approach.

Important: Tracklet frame indices (tracklet.frames) in tracks_2d must be offset by chunk_start when merging. This is critical for correct evaluation across chunk boundaries.
