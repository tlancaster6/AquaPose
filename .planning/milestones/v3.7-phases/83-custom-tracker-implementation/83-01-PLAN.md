    Create `src/aquapose/core/tracking/keypoint_sigmas.py`:
    - `compute_keypoint_sigmas(annotations: list[dict]) -> np.ndarray` — compute per-keypoint sigma from annotation records using COCO methodology: sigma_k = std(||kpt_k - mean_kpt_k|| / sqrt(area)) over all fish instances. Each record has `keypoints` shape (K, 2) and `obb_area` float.

    Create `src/aquapose/core/tracking/keypoint_tracker.py`:

    **_KalmanFilter class** (24-dim constant-velocity):
    - State vector layout: [kp0_x, kp0_y, kp1_x, kp1_y, ..., kp5_x, kp5_y, v_kp0_x, v_kp0_y, ..., v_kp5_x, v_kp5_y] = 12 positions + 12 velocities = 24 dims.
    - Document in code: "CONTEXT.md specifies '60-dim' conceptually (tracking all 6 keypoints' full state). The mathematical dimension is 24: 6 kpts x 2D x 2 (position + velocity)."
    - F matrix: [[I_12, I_12], [0, I_12]] (constant velocity transition)
    - H matrix: [I_12 | 0_12] (extract positions from state)
    - Q matrix: process noise — small for velocity components (e.g., 0.01), moderate for position via velocity coupling
    - R matrix: diagonal 12x12, confidence-scaled per-keypoint: R[2k, 2k] = R[2k+1, 2k+1] = base_R / max(conf_k, epsilon). epsilon=1e-6.
    - `predict() -> np.ndarray`: Returns predicted positions as (6, 2). Advances state via x = F @ x, P = F @ P @ F.T + Q.
    - `update(obs: np.ndarray, confs: np.ndarray) -> None`: Standard Kalman update with confidence-scaled R.
    - `get_state() -> dict` / `from_state(state: dict) -> _KalmanFilter`: Serialize x.tolist(), P.tolist() for chunk handoff safety.

    **OKS + OCM cost functions** (module-level, vectorized):
    - `compute_oks_matrix(pred_kpts, det_kpts, det_confs, det_scales, sigmas) -> np.ndarray` — (N, M) OKS similarity matrix. Use NumPy broadcasting over (N, 1, 6, 2) vs (1, M, 6, 2). Clamp det_scales >= 1.0 to avoid division by zero.
    - `compute_ocm_matrix(pred_headings, det_headings) -> np.ndarray` — (N, M) cosine similarity of heading vectors.
    - `compute_heading(kpts: np.ndarray) -> np.ndarray` — Extract spine1-to-spine3 heading vector (kpts[2] - kpts[4]) and normalize. Keypoint indices: 0=nose, 1=head, 2=spine1, 3=spine2, 4=spine3, 5=tail.
    - `build_cost_matrix(oks, ocm, lambda_ocm=0.2) -> np.ndarray` — (1 - oks) + lambda_ocm * (1 - ocm).

    **_KFTrack dataclass** — per-track state:
    - track_id: int
    - kf: _KalmanFilter
    - time_since_update: int (frames since last match)
    - hit_streak: int (consecutive matched frames)
    - detected_count: int (total matched frames)
    - obs_history: deque of (obs_array, conf_array) tuples, maxlen=5 (for OCR)
    - State: "tentative" or "confirmed"

    **ORU mechanism** — method on _KFTrack:
    - After a coasting track is re-matched, re-initialize the KF from the stored pre-coast state (last confirmed observation) and run predict-update steps to the current frame. Store `_pre_coast_state` (x, P copies) whenever time_since_update transitions from 0 to 1.

    **OCR mechanism** — method on _KFTrack:
    - During coast, scan obs_history for an entry whose OKS with the predicted position exceeds a recovery threshold (0.5). If found, trigger KF update with that observation to stabilize the track.

    **_SinglePassTracker class**:
    - `__init__(camera_id, direction, config)` — direction is "forward" or "backward". config holds max_age, n_init, det_thresh, base_R, lambda_ocm, sigmas.
    - `update(frame_idx, detections: list[Detection]) -> None`:
      1. Filter detections below det_thresh.
      2. Extract keypoints (6, 2) and confidences (6,) from each Detection. Skip detections missing keypoints.
      3. Predict all active tracks.
      4. Build OKS + OCM cost matrix (tracks x detections).
      5. Run scipy.optimize.linear_sum_assignment on cost matrix.
      6. For matched pairs with cost < threshold (1.0): KF update + ORU if recovering from coast. Add frame to builder as "detected".
      7. For unmatched tracks: increment time_since_update, attempt OCR, add to builder as "coasted" if confirmed.
      8. For unmatched detections: create new tentative _KFTrack.
      9. Promote tentative tracks with hit_streak >= n_init to "confirmed".
      10. Cull tracks with time_since_update > max_age.
    - `get_tracklets() -> list[Tracklet2D]`: Return tracklets for confirmed tracks (detected_count >= n_init).
    - `get_state() -> dict` / `from_state(state) -> _SinglePassTracker`: Serialize all active tracks' KF states + metadata.
    - The builder must store keypoint data per frame for Plan 02's merge step. Extend _TrackletBuilder with `keypoints: list[np.ndarray]` and `keypoint_conf: list[np.ndarray]` fields (used internally by the tracker, not part of Tracklet2D output).

    Use the project's `_TrackletBuilder` pattern from `ocsort_wrapper.py` but create an independent `_KptTrackletBuilder` in keypoint_tracker.py to avoid coupling. The builder stores keypoints and confs alongside the standard fields for the merge step.
