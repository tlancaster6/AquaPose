  <name>Task 1: Convert build_yolo_training_data.py to txt+yaml output</name>
  <files>scripts/build_yolo_training_data.py</files>
  <action>
    Modify all three `generate_*` functions in `scripts/build_yolo_training_data.py` to output standard YOLO txt labels + dataset.yaml instead of NDJSON.

    **For each mode (obb, seg, pose), the output structure must be:**
    ```
    {mode}/
      images/
        train/
          {stem}.jpg
        val/
          {stem}.jpg
      labels/
        train/
          {stem}.txt
        val/
          {stem}.txt
      dataset.yaml
    ```

    **Per-image .txt label files:**
    - One line per annotation (per fish in the crop)
    - Space-separated normalized values (the existing `format_*_annotation` functions already return the correct flat list of floats — just join with spaces and write to `.txt`)
    - If an image has multiple annotations, write multiple lines

    **dataset.yaml content by mode:**

    OBB:
    ```yaml
    path: /abs/path/to/obb
    train: images/train
    val: images/val
    nc: 1
    names:
      0: fish
    ```

    Seg:
    ```yaml
    path: /abs/path/to/seg
    train: images/train
    val: images/val
    nc: 1
    names:
      0: fish
    ```

    Pose (includes flip_idx for fliplr augmentation):
    ```yaml
    path: /abs/path/to/pose
    train: images/train
    val: images/val
    nc: 1
    names:
      0: fish
    kpt_shape: [6, 3]
    flip_idx: [0, 1, 2, 3, 4, 5]
    ```

    **Key implementation notes:**
    - The `format_obb_annotation`, `format_pose_annotation`, `format_seg_annotation` functions are correct as-is — they return flat `list[float]`. Only the file-writing infrastructure changes.
    - Create `labels/{split}/` directories alongside `images/{split}/` using `mkdir(parents=True, exist_ok=True)`.
    - Write dataset.yaml using `yaml.dump()` with `sort_keys=False`.
    - Remove ALL NDJSON-writing code (the header/record writing, any `dataset.ndjson` references).
    - Use `yaml` import (PyYAML, already a project dependency).
    - `kpt_shape` for pose: read from the annotation data (the current code already knows n_keypoints). Use identity `flip_idx` as decided in CONTEXT.md.
    - `tmp/convert_all_annotations.py` is NOT in the repo — skip it. Note in summary for user to update their local copy if they have one.
