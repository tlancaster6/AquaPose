  <action>
    Add to `src/aquapose/core/tracking/keypoint_tracker.py`:

    **merge_forward_backward() function**:
    - Takes forward_tracklets (list of internal builder objects with keypoints) and backward_tracklets.
    - Find temporal overlap between each (fwd, bwd) pair. For pairs with >= min_overlap overlapping frames, compute mean OKS in the overlap region using the stored per-frame keypoints.
    - Build OKS cost matrix (N_fwd x N_bwd) and run `scipy.optimize.linear_sum_assignment`.
    - For matched pairs: merge frames. For each overlapping frame, apply resolution policy: detected > coasted; if both detected, keep higher mean keypoint confidence; if both coasted, keep forward.
    - For non-overlapping frames: concatenate.
    - Sort all merged frames by frame index (ascending).
    - Keep unmatched forward/backward tracklets if their length >= min_length.
    - Re-assign monotonically increasing track_ids (0, 1, 2, ...) to all output tracklets.
    - Build Tracklet2D objects from merged builders.

    **interpolate_gaps() function**:
    - Takes a _KptTrackletBuilder (or equivalent internal representation with per-frame keypoints).
    - Scan frames for non-consecutive indices. For each gap <= max_gap_frames:
      - Use `scipy.interpolate.CubicSpline` per-coordinate per-keypoint on the surrounding detected/coasted frames to interpolate keypoint positions at the missing frame indices.
      - Insert interpolated frames with status "coasted" (not a new status value, to avoid breaking downstream consumers per RESEARCH.md Pitfall 6).
      - Interpolated centroids are derived from the interpolated spine1 keypoint (index 2), consistent with the keypoint centroid logic.
      - Interpolated bboxes use linear interpolation of the surrounding bboxes.
    - Return updated builder.

    **KeypointTracker class** (public API, same interface as OcSortTracker):
    - `__init__(camera_id, max_age=15, n_init=3, det_thresh=0.5, base_R=10.0, lambda_ocm=0.2, sigmas=None, max_gap_frames=5, centroid_keypoint_index=2, centroid_confidence_floor=0.3)`:
      - Creates two _SinglePassTracker instances: forward (direction="forward") and backward (direction="backward").
      - Stores all detection list for backward pass replay.
    - `update(frame_idx, detections: list) -> None`:
      - Feed to forward pass tracker immediately.
      - Store (frame_idx, detections) for backward pass replay.
    - `get_tracklets() -> list[Tracklet2D]`:
      - Run backward pass over stored frames in reverse order.
      - Call merge_forward_backward() on both passes' builder-level data.
      - Apply interpolate_gaps() to each merged tracklet.
      - Return final Tracklet2D list.
    - `get_state() -> dict`:
      - After merge, identify active tracks (those whose last frame is the chunk's last frame).
      - Serialize their KF states (x.tolist(), P.tolist()), obs_history, time_since_update, hit_streak.
      - Include next_id counter and camera_id.
      - All values must be JSON-safe (no numpy arrays).
    - `from_state(camera_id, state) -> KeypointTracker`:
      - Reconstruct tracker with restored KF states for active tracks.
      - New chunk starts fresh forward/backward passes but with pre-existing track IDs and KF states seeded into the forward pass.

Forward pass naturally catches track entries, backward pass catches exits. No spatial-edge heuristic needed.
