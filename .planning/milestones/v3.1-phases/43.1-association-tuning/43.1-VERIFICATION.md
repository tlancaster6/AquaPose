# Phase 43.1: Association Tuning -- Verification

## Phase Goal
Association parameters are empirically tuned so that all 9 fish receive multi-camera coverage and can be reconstructed.

## Must-Have Verification

### 1. Evaluation harness supports fast Tier-1-only evaluation via skip_tier2 flag
**Status: PASS**
- `run_evaluation()` in `src/aquapose/evaluation/harness.py` accepts `skip_tier2: bool = False`
- When True, Tier 2 leave-one-out block is skipped entirely
- Returns `Tier2Result(per_fish_dropout={})`
- All existing tests pass with no behavioral change at defaults

### 2. A helper function can generate fixtures from different AssociationConfig values
**Status: PASS**
- `generate_fixture()` exists in `src/aquapose/evaluation/harness.py`
- Accepts `yaml_config_path`, `association_overrides`, and `output_path`
- Patches AssociationConfig via `dataclasses.replace()` before running pipeline
- Re-exported from `aquapose.evaluation` package

### 3. A tuning script sweeps association parameters in priority order and identifies the best config
**Status: PASS**
- `scripts/tune_association.py` exists and was executed against real data
- Script was restructured during interactive tuning to use a joint 2D grid sweep (ray_distance_threshold x score_min) instead of sequential 1D sweeps
- Scoring kernel replaced from binary inlier counting to soft linear scoring
- Sweeps run with both n_frames=15 and n_frames=100

### 4. The winning config produces 9 reconstructed fish per frame (or best achievable yield documented)
**Status: PASS (best achievable yield documented)**
- Multiple sweep runs were executed against real YH project data
- Best result (ray_distance_threshold=0.03, score_min=0.03): 19% yield, 2.90 px mean error, 69.3% singleton rate
- Baseline (current defaults): 18% yield, 2.99 px mean error, 70.1% singleton rate
- Gains are marginal across all parameter combinations tested (18 of 56 combos completed with n_frames=100 before stopping; full sweep with n_frames=15 also showed minimal variation)
- **Root cause**: The association stage is not the primary bottleneck for fish recovery. The singleton rate (~70%) indicates most fish-frame observations only have one camera view, which is an upstream detection/tracking coverage issue rather than an association parameter issue
- **Decision**: Current association defaults are accepted as good enough. No config changes applied.
