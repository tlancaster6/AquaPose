## Decisions Made

- **Tagged split over random split:** Random split treats augmented children as independent samples. When a parent is assigned to val, its 4 augmented children can end up in train (different images, but same keypoints). This corrupts train/val independence. Fix: tag val samples in the database, then use `--split-mode tagged` so augmented children of val parents are automatically excluded from train.
- **val filter: source != pseudo not source == manual:** Corrected pseudo-labels are human-reviewed annotations — they should be in val by default. The old filter (`source == "manual"`) excluded them, making val less representative. New filter only excludes raw pseudo-labels.
- **Combined val sets:** The existing round1_selected 64-image val set was not tagged in the database. Tagged it as "val", yielding a combined 128-image val (64 manual + 64 corrected) — more comprehensive evaluation.
- **4:1 augmentation ratio confirmed important:** First pose training attempt used 2 augments per parent (batch 64), scoring mAP50-95=0.926 — lower than expected. Retrained with 4 augments per parent (batch 128): 0.972. The larger augmentation variety is important for generalization.
- **Batch size 128 for pose training:** Increased from 64 to fully utilize GPU, also paired with larger dataset (4-aug vs 2-aug).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Augmentation leakage in random split**
- **Found during:** Task 1 (dataset assembly)
- **Issue:** `assemble()` random split assigns augmented children independently of their parents. When a parent lands in train and its children in val (or vice versa), val effectively contains near-duplicate images of train samples, breaking train/val independence.
- **Fix:** Tagged 64 round1_selected val samples in the database as "val", then assembled with `--split-mode tagged`. Augmented children of val-tagged parents are automatically excluded from train. Added `--max-aug-per-parent` flag to cap augmentation ratio.
- **Files modified:** `src/aquapose/training/store.py`, `src/aquapose/training/data_cli.py`
- **Verification:** Dataset counts with tagged split showed 2554 train / 128 val with no parent-child split between sets.
- **Committed in:** `bfce249`

**2. [Rule 1 - Bug] Val filter excluded corrected pseudo-labels**
- **Found during:** Task 1 (inspecting store status for assembly)
- **Issue:** `assemble()` used `source == "manual"` to decide val eligibility (when `pseudo_in_val=False`). Corrected pseudo-labels (human-reviewed, source="corrected") were excluded from val, making the val set less representative.
- **Fix:** Changed condition to `source != "pseudo"` so only raw pseudo-labels are excluded; corrected labels pass through.
- **Files modified:** `src/aquapose/training/store.py`
- **Verification:** Combined 128-image val (64 manual + 64 corrected) assembled correctly.
- **Committed in:** `dca77ee`
