# Phase 33.1 Plan 01: Keypoint Training Data Augmentation Summary

**One-liner:** Keypoint-aware augmentation via torchvision.transforms.v2+tv_tensors.KeyPoints with masked MSE loss and ConcatDataset 2x training.

## What Was Built

Added data augmentation to the pose regression training pipeline to improve generalization from the 78-sample annotation dataset.

**`src/aquapose/training/pose.py` changes:**

1. **`_AUGMENT_TRANSFORM`** â€” module-level `v2.Compose` with `RandomHorizontalFlip(p=0.5)`, `RandomAffine(degrees=15, translate=(0.1,0.1), scale=(0.8,1.2))`, and `ColorJitter(brightness=0.3, contrast=0.3, saturation=0.3, hue=0.05)`. All geometric transforms jointly move image + keypoints via `tv_tensors.KeyPoints`.

2. **`KeypointDataset`** â€” Added `augment: bool = False` parameter. Changed `__getitem__` return from 2-tuple to 3-tuple `(image, keypoints, visibility_mask)`. Clean path: visibility from COCO `v > 0` plus OOB check (handles real-data negative-y coordinates per RESEARCH.md Pitfall 6). Augmented path: wraps image as `tv_tensors.Image` (uint8) + keypoints as `tv_tensors.KeyPoints(pixel_coords, canvas_size=(128,128))`, applies `_AUGMENT_TRANSFORM`, detects OOB, retries up to 10 times when `<3 visible`, zeros invisible coords, normalizes to `[0,1]` after transforms.

3. **`_masked_mse_loss`** â€” New function. Expands visibility from `(B, n_kp)` to `(B, n_kp*2)` via `repeat_interleave(2)`, computes squared diff, sums only visible entries, divides by `clamp(min=1.0)` to avoid division-by-zero when all invisible.

4. **`_mean_keypoint_error`** â€” Added optional `visibility: torch.Tensor | None = None` parameter. When provided, uses masked mean distance (backward-compatible with existing callers).

5. **`train_pose`** â€” Updated both pre-split and annotations.json paths to build `ConcatDataset([clean_subset, aug_subset])` for training (2x effective epoch size). Validation remains clean-only. Training loop unpacks 3-tuples and uses `_masked_mse_loss`. Validation loop collects visibility tensors and passes to `_mean_keypoint_error`.

**`tests/unit/training/test_pose.py` additions:**

11 new tests: 3-tuple shape checks (clean+augmented), COCO visibility derivation (partial+OOB edge case), augmentation changes image, OOB coord zeroing, masked MSE loss (zero-for-perfect, all-invisible, ignores invisible), masked vs unmasked validation metric, backward compatibility, and smoke training test.
