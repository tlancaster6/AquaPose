  <files>src/aquapose/training/store.py, src/aquapose/training/run_manager.py, tests/unit/training/test_store.py, tests/unit/training/test_run_manager.py</files>
  <behavior>
    - test_register_model_inserts_row: Register model with run_id, weights_path, model_type, metrics. Verify row in models table.
    - test_register_model_with_dataset: Register model linked to a dataset name. Verify FK.
    - test_list_models_returns_all: Register two models, list returns both ordered by created_at desc.
    - test_get_model_by_run_id: Register model, get by run_id returns correct record.
    - test_update_config_weights_obb: Create config.yaml with detection.weights_path, call update, verify field changed.
    - test_update_config_weights_pose: Create config.yaml with midline.weights_path, call update, verify field changed.
    - test_update_config_preserves_other_fields: Verify other config fields are not modified.
    - test_register_trained_model_autocreates_store: Call register_trained_model when store.db doesn't exist yet, verify store is created and model registered.
  </behavior>
  <action>
    1. Add to SampleStore:
       - `register_model(self, run_id: str, weights_path: str, model_type: str, metrics: dict | None = None, dataset_name: str | None = None, tag: str | None = None) -> None`: INSERT into models table.
       - `list_models(self) -> list[dict]`: SELECT all from models ORDER BY created_at DESC, parse JSON metrics.
       - `get_model(self, run_id: str) -> dict | None`: SELECT by run_id.

    2. Add to run_manager.py:
       - `update_config_weights(config_path: Path, model_type: str, weights_path: Path) -> None`:
         - Read config with yaml.safe_load
         - Map model_type: "obb" -> "detection", "pose" -> "midline"
         - Update `config[section]["weights_path"]` to str(weights_path)
         - Write back with yaml.dump(default_flow_style=False, sort_keys=False)
         - Print green/bold message: "Updated {config_path}: {section}.weights_path = {weights_path}"
       - `register_trained_model(config_path: Path, run_dir: Path, model_type: str, best_weights: Path, dataset_dir: Path | None = None, tag: str | None = None) -> None`:
         - Resolve project dir from config
         - Construct store path: `{project_dir}/training_data/{model_type}/store.db`
         - Open SampleStore (this auto-creates the store if it doesn't exist, per Plan 01's auto-create-on-connect behavior)
         - Read summary.json from run_dir for metrics (if exists)
         - Determine dataset_name from dataset_dir if provided (check if it's a store-managed dataset by looking for the name in the datasets table; if not found, store the dir basename as dataset_name)
         - Call store.register_model(run_id=run_dir.name, weights_path=str(best_weights), model_type=model_type, metrics=metrics, dataset_name=dataset_name, tag=tag)
         - Call update_config_weights(config_path, model_type, best_weights)
  </action>

  </verify>
  <done>Models can be registered in the store with dataset lineage, and config.yaml is auto-updated after training. Store is auto-created if it doesn't exist.</done>
