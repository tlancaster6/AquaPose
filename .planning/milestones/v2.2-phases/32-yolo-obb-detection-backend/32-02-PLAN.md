## Objective

Extend both visualization observers to render OBB polygon overlays when OBB detection data is available, falling back to axis-aligned bounding boxes when it is not.

## Context

@src/aquapose/engine/overlay_observer.py — `Overlay2DObserver._draw_bbox()` to extend with OBB polygon branch
@src/aquapose/engine/tracklet_trail_observer.py — `TrackletTrailObserver._draw_trail()` to extend with OBB polygon at trail head; `FISH_COLORS_BGR` palette
@src/aquapose/segmentation/detector.py — `Detection` dataclass with `obb_points` field
@tests/unit/engine/test_overlay_observer.py — existing overlay tests
@tests/unit/engine/test_tracklet_trail_observer.py — existing trail tests
@.planning/phases/32-yolo-obb-detection-backend/32-RESEARCH.md — Pattern 3: OBB Polygon Overlay
@.planning/phases/32-yolo-obb-detection-backend/32-CONTEXT.md — OBB overlay styling decisions

## Locked Decisions (from CONTEXT.md)

- OBB polygon color matches fish ID color (same palette as tracklet trails, `FISH_COLORS_BGR`)
- OBB polygon REPLACES axis-aligned bounding box — do not draw both
- Labels show fish ID + confidence score (same format as existing AABB overlays)
- No orientation axis line — polygon shape conveys orientation

**1. Modify `_draw_bbox()` in `overlay_observer.py` to handle OBB polygons:**

The method currently draws an axis-aligned rectangle. Extend it to check for `obb_points` on the detection and draw an OBB polygon instead. Per user decision: OBB polygon REPLACES AABB — do not draw both.

Change `_draw_bbox` to also accept an `obb_points` parameter (or better: change the caller to pass the detection object). The cleanest approach:

- Add a new static method `_draw_detection_bbox()` that accepts the full detection object (or the relevant fields):
  ```python
  @staticmethod
  def _draw_detection_bbox(
      frame: np.ndarray,
      bbox: tuple[int, int, int, int],
      color: tuple[int, int, int],
      obb_points: np.ndarray | None = None,
      fish_id: int | None = None,
      confidence: float | None = None,
      thickness: int = 2,
  ) -> None:
  ```

  Implementation:
  - If `obb_points is not None`: draw OBB polygon using `cv2.polylines(frame, [obb_points.astype(np.int32).reshape((-1, 1, 2))], isClosed=True, color=color, thickness=thickness)`. CRITICAL: reshape to `(N, 1, 2)` not `(N, 2)` — cv2.polylines requires this shape.
  - Else: draw axis-aligned rectangle using existing `cv2.rectangle` logic from the current `_draw_bbox`.
  - If `fish_id is not None`: draw label text. Format: `"{fish_id} {conf:.2f}"` when confidence is provided, else just `"{fish_id}"`. Position label near top-left: for OBB use `(int(obb_points[:, 0].min()), int(obb_points[:, 1].min()) - 5)`; for AABB use existing position `(x1, y1 - 5)`.

- Update the call site in `_generate_overlays()` (inside the `if self._show_bbox:` block, around line 252-265). Currently it calls `self._draw_bbox(frames[cam_id], bbox, ...)`. Change to:
  - Extract `obb_points` from the detection: `obb_pts = getattr(det, "obb_points", None)`.

  - Call `self._draw_detection_bbox(frames[cam_id], bbox, self._midline_2d_color, obb_points=obb_pts, fish_id=fish_id_label, confidence=conf)`.

- Keep the old `_draw_bbox` method as-is for backward compatibility (other code may call it). Or remove it if no external callers exist — check first with grep.

- Add `import numpy as np` at the top if not already present.

**1. Add OBB polygon drawing to `TrackletTrailObserver`:**

The observer needs to draw OBB polygons at the trail head position when detection data is available. Per the research, use Option B: access `context.detections` directly from the pipeline context rather than modifying Tracklet2D.

**Approach:** Modify `_generate_trail_videos()` to pass `context.detections` through to the per-camera trail and mosaic generation methods, then draw OBB polygons in `_draw_trail()` and `_draw_trail_scaled()`.

- In `_generate_trail_videos()`:
  - Extract detections: `detections = getattr(context, "detections", None)`.
  - Pass `detections` to `_generate_per_camera_trails()` and `_generate_association_mosaic()` as a new kwarg.

- In `_generate_per_camera_trails()`:
  - Accept new kwarg `detections: list[dict[str, list]] | None = None`.
  - After calling `self._draw_trail(...)`, if `detections is not None` and `frame_idx < len(detections)`:
    - Look up `frame_dets = detections[frame_idx].get(cam_id, [])`.
    - For each active `(tracklet, idx_in_tracklet, fish_id)`, find the matching detection by centroid proximity: compute distance from `tracklet.centroids[idx_in_tracklet]` to each detection's bbox center. Pick the closest one if distance < 50 px.
    - If the matched detection has `obb_points is not None`, draw the OBB polygon using `cv2.polylines(frame, [obb_pts.astype(np.int32).reshape((-1,1,2))], isClosed=True, color=base_color, thickness=2)`.
    - If no `obb_points`, draw the AABB as a rectangle: `x, y, w, h = det.bbox; cv2.rectangle(frame, (x, y), (x+w, y+h), base_color, 2)`.
    - This satisfies VIZ-02: "bounding box overlays (both axis-aligned and OBB when available)".

- In `_generate_association_mosaic()`:
  - Accept `detections` kwarg similarly.
  - After `_draw_trail_scaled()`, draw scaled OBB polygon or AABB for the matched detection. Scale the polygon/bbox coordinates by `scale_x, scale_y` before drawing.

- Add a helper method to avoid code duplication:
  ```python
  @staticmethod
  def _draw_detection_box(
      frame: np.ndarray,
      detection: object,
      color: tuple[int, int, int],
      scale_x: float = 1.0,
      scale_y: float = 1.0,
      thickness: int = 2,
  ) -> None:
  ```
  - Checks `getattr(detection, "obb_points", None)`.
  - If OBB: scale and draw polygon.
  - Else: scale and draw AABB rectangle from `detection.bbox`.

- Add a helper to match detection to tracklet centroid:
  ```python
  @staticmethod
  def _match_detection(
      centroid: tuple[float, float],
      detections: list,
      max_distance: float = 50.0,
  ) -> object | None:
  ```
  - Compute bbox center for each detection: `(bbox[0] + bbox[2]/2, bbox[1] + bbox[3]/2)`.
  - Return the closest detection within `max_distance`, or None.
