# Phase 84.1: Tracker Tuning - Context

**Gathered:** 2026-03-11
**Status:** Ready for planning

<domain>
## Phase Boundary

Tune the custom keypoint tracker (`KeypointTracker`) parameters to reduce over-fragmentation from 42 tracks toward OC-SORT parity (~30 tracks) on the reference clip (e3v83eb, frames 3300-4500, 9 fish). Promote hardcoded thresholds to config, update eval script CLI flags. Minor algorithmic tweaks are in scope if discovered during tuning; new features (appearance embeddings, BYTE pass, sigma calibration) are not.

</domain>

<decisions>
## Implementation Decisions

### Tuning Methodology
- Manual sweep: Claude suggests parameter sets with rationale, user runs the eval script and reports results
- Single camera clip only (e3v83eb, 1200 frames) ‚Äî consistent baseline, fast iteration
- Each run produces both metrics JSON and annotated video for visual ID-switch inspection
- No automated grid search or optimization framework

### Parameter Scope
- **6 parameters in scope:** `base_r`, `lambda_ocm`, `max_age`, `det_thresh`, `match_cost_threshold` (currently `_MATCH_COST_THRESHOLD`), `ocr_threshold` (currently `_OCR_RECOVERY_THRESHOLD`)
- Promote `_MATCH_COST_THRESHOLD` and `_OCR_RECOVERY_THRESHOLD` from hardcoded constants to `TrackingConfig` fields (configurable via YAML/CLI)
- Update eval script to accept all 6 tunable params as CLI flags for easy sweeping
- Keypoint sigmas (`DEFAULT_SIGMAS`) are **deferred** ‚Äî not in scope for this phase

### Evaluation Criteria
- Primary target: track count closest to 9, measured as `|track_count - 9|`
- Success threshold: match OC-SORT (~30 tracks) ‚Äî parity is acceptable for V1
- Minimum detection coverage floor: 90% (currently 93.6%)
- No automated ID-switch metric ‚Äî use annotated video for visual inspection of identity swaps vs fragments
- No ground-truth identity labels exist

### Scope Boundary
- Parameter tuning + config plumbing + minor algorithm tweaks discovered during tuning
- No new features: no appearance embeddings, no BYTE-style secondary pass, no sigma calibration from data
- Best params become new defaults in both `TrackingConfig` dataclass and project YAML (`config.yaml`)
- Fallback if tuning can't reach ~30 tracks: accept best result, document the remaining gap vs OC-SORT, note what algorithmic changes might close it in a future phase

### Claude's Discretion
- Order of parameter exploration (which params to sweep first)
- Specific parameter ranges to suggest per sweep round
- Whether to try parameter combinations or sweep one-at-a-time
- Nature of any minor algorithm tweaks if discovered

</decisions>

<specifics>
## Specific Ideas

- Phase 84 evaluation identified specific tuning directions: increase `base_r` (50-100), reduce `lambda_ocm` (0.05), increase `max_age` (30-60), reduce `_MATCH_COST_THRESHOLD`
- The root cause of over-fragmentation is occlusion reacquisition failure, not temporal drift ‚Äî matching is too strict when fish reappear after crossing paths
- OC-SORT's IoU matching is inherently more permissive than OKS+OCM, which explains the 42 vs 30 gap

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- `scripts/evaluate_custom_tracker.py`: Full eval pipeline ‚Äî runs detection+pose once, caches detections, feeds to both trackers, produces comparison JSON + annotated video. Already has `--max-age`, `--det-thresh`, `--n-init` CLI flags.
- `TrackingConfig` in `engine/config.py`: Frozen dataclass with `base_r`, `lambda_ocm`, `max_gap_frames` already as fields. Needs `match_cost_threshold` and `ocr_threshold` added.
- `eval_tracker_output/comparison_metrics.json`: Current baseline metrics for both trackers.

### Established Patterns
- `TrackingConfig` fields flow through `TrackingStage` ‚Üí `KeypointTracker` constructor ‚Üí `_SinglePassTracker`. Adding new config fields follows this existing wiring.
- `_MATCH_COST_THRESHOLD` and `_OCR_RECOVERY_THRESHOLD` are module-level constants in `keypoint_tracker.py` used by `_SinglePassTracker._step()`. Promoting them means adding constructor params and passing through.

### Integration Points
- `engine/config.py:TrackingConfig` ‚Äî add `match_cost_threshold` and `ocr_threshold` fields
- `core/tracking/stage.py` ‚Äî pass new config fields to `KeypointTracker` constructor
- `core/tracking/keypoint_tracker.py` ‚Äî accept new params in `_SinglePassTracker.__init__()` and `KeypointTracker.__init__()`
- `scripts/evaluate_custom_tracker.py` ‚Äî add CLI flags for all 6 params

</code_context>
