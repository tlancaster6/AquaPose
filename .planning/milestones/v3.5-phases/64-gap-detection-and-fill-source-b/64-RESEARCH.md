# Phase 64: Gap Detection and Fill (Source B) - Research

**Researched:** 2026-03-05
**Domain:** Pseudo-label generation, gap detection via inverse LUT visibility, diagnostic cache traversal
**Confidence:** HIGH

## Summary

Phase 64 extends the pseudo-label system (Phase 63) to detect cameras where a reconstructed fish *should* be visible but was not detected, classify each gap by failure reason, and generate corrective pseudo-labels by reprojecting the 3D reconstruction into those gap cameras. The implementation builds directly on the existing `pseudo_labels.py` and `pseudo_label_cli.py` modules, reusing `compute_confidence_score()`, `reproject_spline_keypoints()`, and label formatting functions. The key new capability is cross-referencing `InverseLUT.ghost_point_lookup()` visibility with `Midline3D.per_camera_residuals` to find gap cameras, then classifying each gap by checking the diagnostic cache's detection, tracking, and midline stages in reverse order.

The output directory structure changes from `pseudo_labels/{obb,pose}/` to `pseudo_labels/{consensus,gap}/{obb,pose}/`, which is a deliberate breaking change. The CLI gains `--consensus` and `--gaps` flags on the existing `generate` command, with `--min-cameras` controlling the contributing-camera floor for gap detection.

**Primary recommendation:** Implement gap detection as a new core function in `pseudo_labels.py` that takes a Midline3D + InverseLUT + PipelineContext and returns gap camera IDs with reasons, then build the gap label generation loop in the CLI as a parallel path to consensus generation.

## User Constraints (from CONTEXT.md)

### Locked Decisions
- Use inverse LUT `ghost_point_lookup()` on the fish's 3D centroid (mean of control points) to determine which cameras should see each fish
- Single centroid query per fish per frame -- no multi-point spline sampling needed
- Gap cameras are those flagged visible by the LUT but NOT in the midline's `per_camera_residuals` (i.e., did not contribute to the reconstruction)
- Contributing cameras are never flagged as gaps, even if their residual is high
- Minimum camera floor: 3 contributing cameras required before gap detection activates (configurable via `--min-cameras`, default 3)
- 3-tier per-frame classification: `no-detection`, `no-tracklet`, `failed-midline`
- Classification logic checks pipeline stages in reverse order (detection, then tracklet, then failed-midline)
- Classification is per-frame, not per-tracklet
- Generate gap-fill labels for ALL gap reasons (no pre-filtering by reason)
- Gap labels use the same `compute_confidence_score()` as Source A
- No discount factor or separate formula for gap labels
- Basic bounds check on reprojected labels: skip if projected spline falls mostly outside image bounds or OBB area is degenerate
- Camera floor (contributing cameras >= 3) is the primary quality gate
- Restructured directory layout: `pseudo_labels/{consensus,gap}/{obb,pose}/{images,labels}/train/`
- Source A output renamed from `pseudo_labels/{obb,pose}/` to `pseudo_labels/consensus/{obb,pose}/`
- Each subset has its own `dataset.yaml` and `confidence.json` sidecar
- Gap sidecar adds per-label fields: `gap_reason` and `n_source_cameras`
- CLI refactored with `--consensus` and `--gaps` flags (at least one required)
- `--min-cameras` flag (default 3) controls the contributing-camera floor
- Gap labels inherit `--lateral-pad` but skip per-camera residual filtering
- Use projected centroid (via RefractiveProjectionModel.project()) for bbox overlap check, not LUT pixel coords

### Pattern 1: Gap Detection Core Function
**What:** A pure function that takes a Midline3D, InverseLUT, and PipelineContext frame data, returns a list of (camera_id, gap_reason) tuples.
**When to use:** For every fish in every frame that has >= min_cameras contributing cameras.
**Example:**
```python
def detect_gaps(
    midline: Midline3D,
    inverse_lut: InverseLUT,
    proj_models: dict[str, RefractiveProjectionModel],
    frame_detections: dict[str, list[Detection]],
    frame_tracks: dict[str, list],  # cam_id -> tracklets active this frame
    min_cameras: int = 3,
) -> list[tuple[str, str]]:
    """Identify gap cameras and classify each by failure reason.

    Returns:
        List of (camera_id, reason) where reason is one of
        'no-detection', 'no-tracklet', 'failed-midline'.
        Empty list if n_cameras < min_cameras.
    """
    # 1. Check camera floor
    if midline.per_camera_residuals is None:
        return []
    contributing = set(midline.per_camera_residuals.keys())
    if len(contributing) < min_cameras:
        return []

    # 2. Compute centroid (mean of control points)
    centroid_3d = midline.control_points.mean(axis=0)  # (3,)

    # 3. Query InverseLUT for visibility
    visible_cams = ghost_point_lookup(inverse_lut, torch.from_numpy(centroid_3d[None]))
    visible_cam_ids = {cam_id for cam_id, _, _ in visible_cams[0]}

    # 4. Gap cameras = visible - contributing
    gap_cam_ids = visible_cam_ids - contributing

    # 5. Classify each gap
    gaps = []
    for cam_id in gap_cam_ids:
        reason = _classify_gap(cam_id, midline, proj_models[cam_id],
                               frame_detections, frame_tracks)
        gaps.append((cam_id, reason))
    return gaps
```

### Pattern 2: Gap Classification (Reverse Stage Check)
**What:** For each gap camera, check pipeline stages in reverse order to determine failure reason.
**When to use:** Inside `detect_gaps()` for each gap camera.
**Example:**
```python
def _classify_gap(
    cam_id: str,
    midline: Midline3D,
    proj_model: RefractiveProjectionModel,
    frame_detections: dict[str, list[Detection]],
    frame_tracks: dict[str, list],
) -> str:
    """Classify a gap by checking stages in reverse order."""
    # Project 3D centroid into this camera for bbox overlap check
    centroid_3d = midline.control_points.mean(axis=0)
    px, valid = proj_model.project(torch.from_numpy(centroid_3d[None].astype(np.float32)))
    if not valid[0]:
        return "no-detection"  # Can't even project -- treat as no-detection

    cx, cy = float(px[0, 0].cpu()), float(px[0, 1].cpu())

    # 1. Check detections: does any bbox contain the projected centroid?
    cam_dets = frame_detections.get(cam_id, [])
    matching_det = None
    for det in cam_dets:
        x, y, w, h = det.bbox
        if x <= cx <= x + w and y <= cy <= y + h:
            matching_det = det
            break

    if matching_det is None:
        return "no-detection"

    # 2. Check tracklets: does any tracklet cover this detection in this frame?
    cam_tracks = frame_tracks.get(cam_id, [])
    has_tracklet = False
    for tracklet in cam_tracks:
        # Check if tracklet is active and detected (not coasted) at this frame
        if midline.frame_index in tracklet.frames:
            idx = tracklet.frames.index(midline.frame_index)
            if tracklet.frame_status[idx] == "detected":
                has_tracklet = True
                break

    if not has_tracklet:
        return "no-tracklet"

    # 3. Detection and tracklet exist but camera didn't contribute
    return "failed-midline"
```

### Anti-Patterns to Avoid
- **Importing engine modules at top level in training/:** Use `importlib.import_module()` pattern established in Phase 63 to avoid AST boundary violations.
- **Using LUT pixel coords for bbox overlap:** The CONTEXT.md explicitly states to use `RefractiveProjectionModel.project()` for the centroid reprojection during gap classification, not the LUT voxel-resolution pixel coords.
- **Filtering gaps by reason before label generation:** The decision is to generate labels for ALL gap reasons. Phase 65 can filter by reason during dataset assembly.
- **Applying per-camera residual threshold to gap labels:** Gap cameras have no residual in `per_camera_residuals`. The `--max-camera-residual` flag only applies to consensus labels.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| 3D-to-2D projection | Custom projection math | `RefractiveProjectionModel.project()` | Handles Snell's law refraction correctly |
| Visibility lookup | Custom voxel grid search | `ghost_point_lookup()` from `calibration/luts.py` | Pre-computed InverseLUT with O(1) grid lookup |
| Spline evaluation + reprojection | Manual B-spline evaluation | `reproject_spline_keypoints()` from `pseudo_labels.py` | Handles knot domain mapping, BSpline evaluation, projection |
| OBB computation | Custom bounding box logic | `pca_obb()` + `format_obb_annotation()` from `geometry.py` | PCA-based OBB with proper corner ordering |
| Confidence scoring | New confidence formula | `compute_confidence_score()` from `pseudo_labels.py` | Same fish-level formula, no discount for gaps |
| Diagnostic cache loading | Custom pickle loading | `load_run_context()` from `evaluation/runner.py` | Handles chunk merging, frame offsets, fingerprint checks |

## Common Pitfalls

### Pitfall 1: CUDA Tensor Safety
**What goes wrong:** Calling `.numpy()` on CUDA tensors from `RefractiveProjectionModel.project()`.
**Why it happens:** Projection model may run on GPU.
**How to avoid:** Always use `.cpu().detach().numpy()` pattern (established in CLAUDE.md).
**Warning signs:** `RuntimeError: Can't call numpy() on Tensor that requires grad`.

### Pitfall 2: InverseLUT Loading Requires Calibration Path + LUT Config
**What goes wrong:** InverseLUT is not part of the diagnostic cache. Trying to access it from `PipelineContext` will fail.
**Why it happens:** LUTs are pre-generated infrastructure, not pipeline stage output.
**How to avoid:** Load InverseLUT separately using `load_inverse_luts(calibration_path, lut_config)`. This requires constructing a `LutConfigLike` object from the frozen config. Use the `_LutConfigFromDict` pattern from Phase 62.
**Warning signs:** `AttributeError` when looking for LUT data in the context object.

### Pitfall 3: Tracklet Frame Lookup Performance
**What goes wrong:** Doing `midline.frame_index in tracklet.frames` is O(N) for each tracklet on each gap check.
**Why it happens:** `tracklet.frames` is a tuple (immutable), not a set.
**How to avoid:** Pre-build a frame-to-tracklet index per camera before the main loop. For each camera, build a dict mapping frame_idx to list of active tracklets.
**Warning signs:** Quadratic scaling on large diagnostic caches (9450 frames x 12 cameras x ~4 fish).

### Pitfall 4: Detection Data Structure Alignment
**What goes wrong:** `context.detections` is indexed by frame (list), then by camera_id (dict), then contains a list of Detection objects. Misindexing gives wrong detections.
**Why it happens:** The `detections` field is `list[dict[str, list[Detection]]]` -- three levels of nesting.
**How to avoid:** Use `context.detections[frame_idx].get(cam_id, [])` consistently.
**Warning signs:** Getting a dict when expecting a list, or KeyError on camera IDs.

### Pitfall 5: Tracklet-to-Detection Matching for Classification
**What goes wrong:** The `no-tracklet` classification needs to check whether any tracklet covers the *matching detection* in the gap frame, not just whether any tracklet exists.
**Why it happens:** Multiple detections can exist in the same camera/frame. The tracklet check should verify a tracklet covers the detection that matches the projected centroid.
**How to avoid:** First find the matching detection (bbox contains projected centroid), then check if any tracklet's centroid at that frame is close to the detection centroid. Use the existing `_CENTROID_MATCH_TOLERANCE_PX = 5.0` pattern from `evaluation/runner.py`.
**Warning signs:** Incorrectly classifying `no-tracklet` as `failed-midline` because an unrelated tracklet exists in that camera/frame.

### Pitfall 6: Image Sharing Between Consensus and Gap Generation
**What goes wrong:** When both `--consensus` and `--gaps` are passed, reading video frames twice wastes I/O.
**Why it happens:** The current generate loop reads frames per frame_idx when labels exist.
**How to avoid:** If both flags are passed, process consensus and gap labels for the same frame together, reading the frame image once. Write to separate directories.
**Warning signs:** Double I/O time when both flags are active.

### Bounds Check for Gap Labels
```python
# Source: CONTEXT.md decision -- skip if projected spline falls mostly outside bounds
def _passes_bounds_check(
    keypoints_2d: np.ndarray,
    visibility: np.ndarray,
    img_w: int,
    img_h: int,
) -> bool:
    """Check that most visible keypoints fall within image bounds."""
    vis_pts = keypoints_2d[visibility]
    if len(vis_pts) < 2:
        return False
    in_bounds = (
        (vis_pts[:, 0] >= 0) & (vis_pts[:, 0] < img_w) &
        (vis_pts[:, 1] >= 0) & (vis_pts[:, 1] < img_h)
    )
    return in_bounds.sum() >= len(vis_pts) * 0.5  # at least 50% in bounds
```
