Association evaluator input note:
- Association evaluator accepts `list[MidlineSet]` where `MidlineSet = dict[int, dict[str, Midline2D]]`
- PipelineContext stores `tracklet_groups: list[TrackletGroup]` NOT MidlineSet directly
- However, the association stage cache is loaded as the LATEST PipelineContext snapshot at that point
- The midline stage's annotated_detections field contains the Midline2D data
- For association evaluation, the evaluator needs data from BOTH association output (tracklet_groups for yield/singleton) AND midline output (for camera coverage per fish)
- **Key insight from 47-02-PLAN:** evaluate_association actually iterates midline_sets where each MidlineSet is `dict[int, dict[str, Midline2D]]` — fish_id to camera_id to Midline2D. This is derived from the midline stage output, NOT the association stage directly.
- **Practical approach:** Use the LAST available cache (reconstruction or midline) that has the most data. The midline cache's annotated_detections can be transformed into MidlineSet format. OR: if Phase 47's evaluate_association was designed around TrackletGroup directly, adapt accordingly.
- **Safest approach for the executor:** Read Phase 47's actual evaluate_association signature from the installed code at implementation time. The PLAN specifies the interface; if it changed, adapt.
