  <files>src/aquapose/training/data_cli.py, src/aquapose/cli.py, tests/unit/training/test_data_cli.py</files>
  <behavior>

    - test_import_with_augment_creates_variants: Import single sample with --augment, verify augmented children exist in store with parent_id
    - test_import_reports_counts: Import mixed new+duplicate samples, verify output shows imported/upserted/skipped counts
    - test_import_with_batch_id: Import with --batch-id, verify import_batch_id in sample metadata
    - test_convert_coco_to_yolo: Create minimal COCO JSON with one image+annotation, run convert command, verify YOLO-format output directory created
    - test_data_group_registered: Verify `aquapose data --help` shows import and convert subcommands
  </behavior>
  <action>
    Create `src/aquapose/training/data_cli.py` with:

    1. `@click.group("data") def data_group()` — "Manage training data stores."

    2. `@data_group.command("import")` with options:
       - `--config` (required, Path exists) — project config for path resolution
       - `--store` (required, Choice["obb", "pose"]) — which store to import into
       - `--source` (required, Choice["manual", "corrected", "pseudo"]) — data source type
       - `--input-dir` (required, Path exists) — YOLO-format directory with images/ and labels/
       - `--augment` (flag) — generate elastic augmentation variants on import
       - `--augment-count` (int, default 4) — number of variants per sample when --augment is set
       - `--augment-angle-range` (float, default 15.0) — max deformation angle in degrees
       - `--batch-id` (str, optional) — shared import batch ID for cross-type tracking
       - `--metadata-json` (str, optional) — JSON string of additional metadata to attach

       Implementation:
       - Resolve project dir from config, construct store path: `{project_dir}/training_data/{store}/store.db`
       - Create SampleStore instance
       - Scan input-dir for image files (glob `images/**/*.jpg` + `images/**/*.png`)
       - For each image, find matching label by stem in labels/ directory
       - Call store.import_sample() for each pair
       - If --augment and source != "pseudo": for each newly imported sample (action == "imported"), call elastic_deform.generate_variants() to create variants, then call store.add_augmented() for each variant
       - Print summary: N imported, N upserted, N skipped, N augmented variants created
       - Warn if augmented variants were cascade-deleted during upserts

    3. `@data_group.command("convert")` with options:
       - `--coco-file` (required, Path exists) — path to COCO keypoints JSON
       - `--images-dir` (required, Path exists) — directory containing source images
       - `--output-dir` (required, Path) — output directory for YOLO-format data
       - `--type` (required, Choice["obb", "pose", "both"]) — what to generate
       - `--n-keypoints` (int, default 6) — number of keypoints for pose conversion

       Implementation:
       - Replaces functionality of scripts/build_yolo_training_data.py
       - For "obb": Read COCO annotations, compute OBBs using geometry.py functions, write YOLO-OBB format labels
       - For "pose": Use coco_interchange.coco_to_yolo_pose() to convert
       - For "both": Do both, sharing a batch ID in a sidecar for later cross-type import tracking
       - Output is standard YOLO directory structure ready for `aquapose data import`
       - Print summary of converted annotations

    4. Wire into main CLI: In `src/aquapose/cli.py`, add `from aquapose.training.data_cli import data_group` and `cli.add_command(data_group)`

    5. Delete `scripts/build_yolo_training_data.py` after convert command is working.

    Note on augmentation integration: The elastic_deform.generate_variants() function works on pose (crop-space) data. For --augment with pose store, call it directly. For OBB store, augmentation is not applicable (full-frame images). Print info message if --augment used with --store obb.
  </action>
