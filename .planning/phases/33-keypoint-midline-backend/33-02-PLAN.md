    def _weighted_triangulate_rays(
        origins: torch.Tensor,     # (N, 3)
        directions: torch.Tensor,  # (N, 3)
        weights: torch.Tensor,     # (N,)
    ) -> torch.Tensor:
        """Weighted DLT triangulation: minimize ||diag(w) A x||^2.

        Each camera contributes 2 rows to the DLT A-matrix. Rows are scaled by
        the corresponding weight before SVD, implementing standard weighted
        least squares. When all weights are 1.0, this is equivalent to
        unweighted triangulate_rays().

        Args:
            origins: Ray origins, shape (N, 3).
            directions: Ray directions (unit vectors), shape (N, 3).
            weights: Per-camera weights, shape (N,). Typically sqrt(confidence).

        Returns:
            Triangulated 3D point, shape (3,).
        """
    ```

    Implementation: Build the same DLT A-matrix as `triangulate_rays` does in `calibration/projection.py`, but scale each camera's pair of rows by `weights[i]` before running SVD. The DLT formulation for ray-ray intersection:

    For each camera i with origin O_i and direction D_i, the two rows are:
    - Row 2i:   `[D_i[1]*x - D_i[0]*y, D_i[2]*x - D_i[0]*z, ...]` — this is the standard cross-product DLT.

    Actually, look at how `triangulate_rays` works in `calibration/projection.py`. Read that function first to replicate the A-matrix construction. The key pattern: for each ray (origin, direction), construct two constraint equations from the cross product `direction x (point - origin) = 0`, which gives 3 equations (of which 2 are independent). Build A-matrix, apply weights to row pairs, then SVD solve.

    A simpler approach that's standard in multi-view geometry: for each ray, the point P should satisfy `(P - O_i) x D_i = 0`. Expanding the cross product gives 3 linear equations per camera (pick any 2 linearly independent ones). Stack into A-matrix, scale by weights, solve via SVD (last column of V^T).

    **IMPORTANT:** Read `src/aquapose/calibration/projection.py`, find `def triangulate_rays`, and replicate the exact same A-matrix construction. Then add row-wise weight scaling. This ensures the unweighted case (all weights=1.0) produces identical results.

    def _weighted_chamfer_distance_2d(
        proj: torch.Tensor,
        obs: torch.Tensor,
        weights: torch.Tensor,
    ) -> torch.Tensor:
        """Confidence-weighted symmetric 2D chamfer distance.

        The obs->proj direction uses per-point weights (sqrt confidence).
        The proj->obs direction remains unweighted (projected spline points
        have no per-point confidence).

        Args:
            proj: Projected spline points, shape (M, 2), float32.
            obs: Observed skeleton points, shape (N, 2), float32.
            weights: Per-observation-point weights, shape (N,), float32.
                Typically sqrt(confidence) values.

        Returns:
            Scalar weighted chamfer distance tensor.
        """
    ```

    Implementation:
    - Compute pairwise distances `dists = torch.cdist(proj, obs)` — shape (M, N).
    - proj->obs (unweighted): `min_proj_to_obs = dists.min(dim=1).values.mean()` — same as before.
    - obs->proj (weighted): `min_obs_to_proj = dists.min(dim=0).values` — shape (N,). Weighted mean: `(min_obs_to_proj * weights).sum() / (weights.sum() + 1e-8)`.
    - Return `(min_proj_to_obs + weighted_obs_to_proj) / 2.0`.
    - Handle empty tensors: return zero scalar if either set is empty (same as _chamfer_distance_2d).
