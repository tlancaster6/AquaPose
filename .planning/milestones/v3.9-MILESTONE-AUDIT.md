### Issues (2/10)

**INT-01 (low):** `Midline3DWriter.__init__` defaults `n_sample_points=15` — mismatches system default of 6. The orchestrator always passes `config.n_sample_points`, so runtime behavior is correct. Risk: direct construction without explicit arg creates wrong HDF5 layout.

**INT-02 (medium):** `evaluation/runner.py:631` and `evaluation/tuning.py:162` hardcode `n_points=15` in `_keypoints_to_midline()` calls. The evaluation path builds 15-point Midline2D objects while the pipeline now produces 6-point ones. `compute_per_point_error()` silently skips all raw-keypoint Midline3D because `6 != 15`.
