  <action>
Assemble both training datasets from the existing data stores using the `data assemble` CLI. Per user decision, use all-source stratified val (manual + corrected pseudo-labels in both train and val), NOT manual-only val.

**Step 1: Check current store status**
Run `hatch run aquapose --project ~/aquapose/projects/YH data status` to confirm what samples are available in both OBB and pose stores. Note total counts, sources, and augmented counts.

**Step 2: Assemble OBB dataset**
```bash
hatch run aquapose --project ~/aquapose/projects/YH data assemble \
  --store obb \
  --name production_retrain \
  --pseudo-in-val \
  --val-fraction 0.2 \
  --seed 42
```
Key flags:
- `--pseudo-in-val`: Allows pseudo-labels in val split (all-source val per user decision)
- No `--source` filter: includes all sources (manual + corrected pseudo-labels)
- `--split-mode random` (default): random stratified split

**Step 3: Assemble Pose dataset with augmentation**
First assemble the base pose dataset:
```bash
hatch run aquapose --project ~/aquapose/projects/YH data assemble \
  --store pose \
  --name production_retrain \
  --pseudo-in-val \
  --val-fraction 0.2 \
  --seed 42
```

Note: Elastic augmentation is already in the store from import time (curated+aug pattern from Round 1). Check if augmented variants exist in the store status output. If augmented variants ARE present in the pose store, they will be included automatically in the assembled dataset. If NOT present, run the import with `--augment --augment-count 4 --augment-angle-range 15` on the pose data first.

**Step 4: Verify datasets**
Count train/val images in both assembled datasets. Report counts to confirm assembly succeeded.
