<objective>
Create `aquapose viz` CLI subcommands that generate overlay, animation, and trail visualizations from cached diagnostic data.

Purpose: Visualization was previously coupled to the pipeline via observers (Overlay2DObserver, Animation3DObserver, TrackletTrailObserver). This plan moves visualization to post-run commands that operate on cached data, enabling re-rendering without re-running the pipeline and supporting multi-chunk runs.

Output: `aquapose viz overlay|animation|trails|all <run_dir>` CLI commands that load chunk caches and produce videos/HTML in `{run_dir}/viz/`.
</objective>

1. Create `src/aquapose/evaluation/viz/` package with `__init__.py`.

2. **overlay.py** — `generate_overlay(run_dir: Path, output_dir: Path | None = None) -> Path`:
   - Load all chunk caches using shared loader (from Plan 02)
   - Load config.yaml from run_dir to get calibration_path, video_dir
   - Construct VideoFrameSource from config paths
   - Iterate all frames across all chunks, drawing 2D midlines and reprojected 3D midlines
   - Produce a single continuous mosaic video spanning all chunks
   - Output to `{output_dir or run_dir}/viz/overlay_mosaic.mp4`
   - Reuse `Overlay2DObserver._reproject_3d_midline`, `_draw_midline`, `_draw_midline_points`, `_build_mosaic`, `_draw_detection_bbox` as standalone functions or by extracting them. The simplest approach: copy the core rendering logic from Overlay2DObserver into this module as module-level functions, since the observer coupling (on_event, PipelineComplete) is removed.
   - Fish colors: deterministic from global fish ID — `palette[fish_id % len(palette)]`.
   - If video path from config.yaml is unreachable, fall back to synthetic black frames.

3. **animation.py** — `generate_animation(run_dir: Path, output_dir: Path | None = None) -> Path`:
   - Load all chunk caches, merge midlines_3d across chunks
   - Build a single Plotly figure with unified timeline (frame indices are global)
   - Fish colors: deterministic from global fish ID — `palette[fish_id % len(palette)]`
   - Output to `{output_dir or run_dir}/viz/animation_3d.html`
   - Reuse Animation3DObserver._build_figure and _eval_spline logic.
   - Chunk boundaries should be invisible to the viewer (continuous scrubber).

4. **trails.py** — `generate_trails(run_dir: Path, output_dir: Path | None = None) -> Path`:
   - Load all chunk caches, merge tracks_2d and tracklet_groups across chunks
   - Produce continuous per-camera trail videos and association mosaic
   - Fish colors: deterministic from global fish ID — `palette[fish_id % len(palette)]`. Use `FISH_COLORS_BGR` from tracklet_trail_observer.py (move to visualization/colors.py or import from trails module).
   - Output to `{output_dir or run_dir}/viz/tracklet_trails_{cam_id}.mp4` and `viz/association_mosaic.mp4`
   - Reuse TrackletTrailObserver rendering logic.

5. **__init__.py** — Export `generate_overlay`, `generate_animation`, `generate_trails`, and a convenience `generate_all(run_dir, output_dir)` that calls all three, catching and logging exceptions per visualization (skip on failure).

6. **tests** — Unit tests with minimal fixtures:
   - Test that generate_all calls each sub-generator and catches failures gracefully
   - Test that generate_overlay with an empty cache returns without crash
   - Test file paths are correct ({run_dir}/viz/...)
