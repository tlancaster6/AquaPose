Replace the body of `score_tracklet_pair()` (everything after `lut_b = forward_luts[cam_b]`) with the two-phase batched implementation. **Do NOT change the function signature or docstring.**

**Implementation (per locked decisions in CONTEXT.md):**

1. **Helper: `_batch_score_frames()`** (optional private function, Claude's discretion) or inline the logic. Either way, the pattern is:

2. **Single-phase path** (when `t_shared <= config.early_k`):
   - Stack all shared-frame centroids into `(N, 2)` numpy arrays:
     ```python
     idx_a = [frames_a[f] for f in shared_frames]
     idx_b = [frames_b[f] for f in shared_frames]
     cents_a = np.array([tracklet_a.centroids[i] for i in idx_a], dtype=np.float64)
     cents_b = np.array([tracklet_b.centroids[i] for i in idx_b], dtype=np.float64)
     ```
   - Convert to torch float32 for `cast_ray`: `pix_a = torch.tensor(cents_a, dtype=torch.float32)`
   - Single `cast_ray` call per camera: `origins_a, dirs_a = lut_a.cast_ray(pix_a)`
   - Convert once to numpy float64: `oa = origins_a.cpu().numpy().astype(np.float64)`
   - Call `ray_ray_closest_point_batch(oa, da, ob, db)` — returns `(N,)` distances
   - Vectorized soft kernel: `inlier = dists < config.ray_distance_threshold`, `contributions = np.where(inlier, 1.0 - dists / config.ray_distance_threshold, 0.0)`, `score_sum = float(contributions.sum())`
   - Early check: `if score_sum == 0.0: return 0.0`
   - Compute and return `f * w` as before

3. **Two-phase path** (when `t_shared > config.early_k`):
   - Phase 1: batch first `config.early_k` frames using same pattern as above
   - Early check: `if score_sum == 0.0: return 0.0`
   - Phase 2: batch remaining `shared_frames[config.early_k:]` frames
   - Add Phase 2 contributions to `score_sum`
   - Compute and return `f * w`

4. **Key constraints** (from CONTEXT.md locked decisions):
   - Use `ray_ray_closest_point_batch()` for BOTH phases — never call the scalar version
   - One `cast_ray` call per camera per phase (not per frame)
   - One `.cpu().numpy()` conversion per batched call (not per frame)
   - `score_sum == 0.0` exact comparison (not tolerance) — contributions excluded by threshold are exactly 0.0 via `np.where`
   - `score_tracklet_pair()` public signature unchanged

5. **Do NOT modify** `score_all_pairs()`, `ray_ray_closest_point()`, or `ray_ray_closest_point_batch()`.
