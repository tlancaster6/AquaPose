**Replace the existing `init_config` command** with a new version:

New signature: `aquapose init-config <name> [--synthetic]`
- `name` is a positional argument (Click `@click.argument("name")`)
- `--synthetic` is a boolean flag (default False)

New behavior:
1. Resolve project directory: `project_dir = Path("~/aquapose/projects").expanduser() / name`
2. If `project_dir` exists, raise `click.ClickException(f"'{project_dir}' already exists.")`
3. Create `project_dir` and subdirectories: `runs/`, `models/`, `geometry/`, `videos/`
4. Generate `config.yaml` using an ordered dict (NOT `dataclasses.asdict`) with `yaml.dump(sort_keys=False)`:

```python
import yaml

data: dict[str, Any] = {}
# --- Paths (edited first by users) ---
data["project_dir"] = str(project_dir)
data["video_dir"] = "videos/"
data["calibration_path"] = "geometry/calibration.json"
data["output_dir"] = "runs/"
# --- Core parameters ---
data["n_animals"] = "SET_ME"  # required — must be an integer
# --- Optional (auto-detected if absent) ---
# data["device"] is intentionally omitted — auto-detected
# data["stop_frame"] is intentionally omitted — default None (all frames)
# --- Detection ---
data["detection"] = {"detector_kind": "yolo", "model_path": "models/best.pt"}
# --- Midline ---
data["midline"] = {"weights_path": "models/unet_best.pth"}
# --- Synthetic (only with --synthetic) ---
if include_synthetic:
    data["synthetic"] = {"frame_count": 30, "noise_std": 0.0, "seed": 42}
    # Note: no fish_count here — uses top-level n_animals
```

5. Write the YAML with brief comment header:
```python
header = "# AquaPose pipeline config\n# See documentation for advanced options\n\n"
yaml_content = yaml.dump(data, default_flow_style=False, sort_keys=False)
(project_dir / "config.yaml").write_text(header + yaml_content)
```

6. Print success: `click.echo(f"Project created at {project_dir}")`

**Important:** The generated YAML uses `"SET_ME"` for `n_animals` as a placeholder. This is a string that will fail `load_config()` validation (which expects an int > 0) — forcing the user to set it. Alternative: use a YAML comment. Choose whichever is cleaner. Using a comment like `n_animals:  # REQUIRED — set to number of fish` with no value would produce `n_animals: null` which also fails validation cleanly.

**Locked decision:** Field order is project_dir → paths → n_animals/device → stage params. The `device` field is intentionally ABSENT from generated YAML (auto-detected). The `--synthetic` section does NOT include `fish_count`.

**Path resolution in load_config()** (`src/aquapose/engine/config.py`):

After YAML and CLI layers are merged into `top_kwargs`, `det_kwargs`, `mid_kwargs`, etc., but BEFORE constructing dataclass instances, add project_dir path resolution:

```python
project_dir_str = top_kwargs.get("project_dir", "")
if project_dir_str:
    project_dir = Path(project_dir_str).expanduser().resolve()
    # Resolve top-level path fields
    for key in ("video_dir", "calibration_path", "output_dir"):
        val = top_kwargs.get(key, "")
        if val and not Path(val).is_absolute():
            top_kwargs[key] = str(project_dir / val)
    # Resolve sub-config path fields
    for kwargs, field_names in [
        (det_kwargs, ["model_path"]),
        (mid_kwargs, ["weights_path"]),
    ]:
        for fname in field_names:
            val = kwargs.get(fname, "")
            if val and not Path(val).is_absolute():
                kwargs[fname] = str(project_dir / val)
```

This runs BEFORE run_id/output_dir resolution — so output_dir gets resolved relative to project_dir if it's a relative path.

**Note:** The `resolved_output_dir` fallback (`_default_output_dir`) should remain unchanged — it only triggers when `output_dir` is not in `top_kwargs` at all.
