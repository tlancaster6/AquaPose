Output: Batched detection with OOM resilience, backward-compatible single-frame `detect()` preserved.

- If `not frames`: return `[]`
- Call `self._detector._model.predict(frames, conf=self._detector._conf, iou=self._detector._iou, verbose=False, batch=len(frames))`
- Parse each Result using the same box-parsing logic as `YOLODetector.detect()` (with padding). Extract shared parsing into a helper to avoid duplication.
- Return `list[list[Detection]]` in positional correspondence.

Keep existing `detect()` methods unchanged for backward compatibility.

Refactor `DetectionStage.__init__()`:
- Accept an optional `detection_batch_frames: int = 0` parameter.
- Create a `BatchState` instance as `self._batch_state = BatchState()`.
- Store `self._batch_size = detection_batch_frames`.

Refactor `DetectionStage.run()` inner loop:
- Instead of iterating cameras and calling `self._detector.detect(frames[cam_id])` per camera:
  1. Collect camera frames: `frame_list = []` and `cam_order = []`. For each cam_id in camera_ids, if cam_id in frames, append `(cam_id, frames[cam_id])` to build the ordered list. Track missing cameras separately.
  2. Call `predict_with_oom_retry(self._detector.detect_batch, [f for _, f in frame_list_with_ids], self._batch_size, self._batch_state)` -- this wraps detect_batch with OOM retry.
     - BUT: `predict_with_oom_retry` expects `predict_fn(chunk) -> list`. `detect_batch` returns `list[list[Detection]]`. Each element of the outer list corresponds to one frame. So `predict_fn = self._detector.detect_batch` and `inputs = [frame for cam_id, frame in ordered_items]`. The returned list from predict_with_oom_retry is `list[list[Detection]]` in positional correspondence.
  3. Map results back: `frame_dets = {}`. For each `(i, (cam_id, _))` in enumerate(ordered_items), set `frame_dets[cam_id] = batch_results[i]`. For missing cameras, set `frame_dets[cam_id] = []`.
- After the main loop, if `self._batch_state.oom_occurred`, log a recommendation: `logger.info("Detection batch size was reduced to %d due to CUDA OOM. Consider setting detection.detection_batch_frames=%d in config.", self._batch_state.effective_batch_size, self._batch_state.effective_batch_size)`

Update `build_stages()` in `engine/pipeline.py` to pass `detection_batch_frames=config.detection.detection_batch_frames` to `DetectionStage`.

Update existing detection stage tests to verify the batched path works with a mock backend. Add a test confirming that `detect_batch` is called once per timestep rather than `detect` being called per camera.
