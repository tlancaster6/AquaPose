<task type="auto">
  <name>Task 1: Convert, import, and assemble manual annotations</name>
  <files>No source files modified -- workflow execution only</files>
  <action>
    Run the data management workflow end-to-end using the AquaPose CLI. All commands use the project context (`--project ~/aquapose/projects/YH` or equivalent).

    1. **Convert COCO to YOLO with temporal split:**
       ```
       aquapose --project ~/aquapose/projects/YH data convert \
         --coco-file ~/aquapose/projects/YH/training_data/raw/annotations.json \
         --images-dir ~/aquapose/projects/YH/training_data/raw/images \
         --output-dir ~/aquapose/projects/YH/training_data/converted \
         --type both \
         --split-mode temporal
       ```
       Verify output: check that converted dir has obb/ and pose/ subdirs each with train/ and val/ containing the expected sample counts. With 10 frame indices and 20% val, expect last 2 frame indices in val.

    2. **Import OBB into store:**
       ```
       aquapose --project ~/aquapose/projects/YH data import \
         --store obb \
         --source manual \
         ~/aquapose/projects/YH/training_data/converted/obb
       ```
       This will import train and val samples. Samples from val/ get "val" tag automatically.

    3. **Import pose into store with elastic augmentation:**
       ```
       aquapose --project ~/aquapose/projects/YH data import \
         --store pose \
         --source manual \
         --augment \
         ~/aquapose/projects/YH/training_data/converted/pose
       ```
       This imports pose samples and generates 4 elastic-augmented variants per sample (5-15 degree range). Val samples get "val" tag.

    4. **Verify store contents:**
       ```
       aquapose --project ~/aquapose/projects/YH data status
       ```
       Check that OBB store shows manual samples and pose store shows manual samples + augmented variants. Check val-tagged sample counts match expectations.

    5. **Assemble OBB dataset with tagged split:**
       ```
       aquapose --project ~/aquapose/projects/YH data assemble \
         --store obb --name baseline --split-mode tagged
       ```

    6. **Assemble pose dataset with tagged split:**
       ```
       aquapose --project ~/aquapose/projects/YH data assemble \
         --store pose --name baseline --split-mode tagged
       ```

    7. **Verify assembled datasets:**
       Check that baseline dataset dirs exist with train/val splits matching the val-tagged samples. Confirm dataset.yaml files are correct.

    Note: If any existing store.db files exist from prior experiments, they may need to be backed up or removed first. Check before importing to avoid unintended dedup behavior.
  </action>
  <verify>
    <automated>aquapose --project ~/aquapose/projects/YH data status</automated>
  </verify>
  <done>
    - OBB store has manual annotations with correct counts
    - Pose store has manual annotations + elastic augmented variants
    - Val samples are tagged with "val" in both stores
    - Assembled "baseline" datasets exist for both OBB and pose with correct train/val splits
  </done>

<task type="auto">
  <name>Task 2: Train and register baseline models</name>
  <files>No source files modified -- workflow execution only</files>
  <action>
    Train baseline OBB and pose models from the assembled datasets using the training CLI.

    1. **Train baseline OBB model:**
       ```
       aquapose --project ~/aquapose/projects/YH train obb \
         --data-dir ~/aquapose/projects/YH/training_data/obb/datasets/baseline \
         --tag baseline \
         --epochs 100 \
         --mosaic 0.3 \
         --patience 100
       ```
       Note: mosaic=0.3 is now the CLI default per Plan 01, but specify explicitly for clarity.
       This will take ~30-60 minutes on GPU. The training CLI auto-registers the model in the store after completion.

    2. **Train baseline pose model:**
       ```
       aquapose --project ~/aquapose/projects/YH train pose \
         --data-dir ~/aquapose/projects/YH/training_data/pose/datasets/baseline \
         --tag baseline \
         --epochs 100 \
         --mosaic 0.1 \
         --imgsz 128 \
         --rect \
         --patience 100
       ```
       Note: mosaic=0.1, imgsz=128, rect=True are now the CLI defaults per Plan 01, but specify explicitly for clarity.
       This will take ~20-40 minutes on GPU. Auto-registers after completion.

    3. **Verify registration:**
       ```
       aquapose --project ~/aquapose/projects/YH data status
       ```
       Both stores should show 1 model each, tagged as "baseline".

    4. **Verify project config updated:**
       Check that `~/aquapose/projects/YH/config.yaml` has been updated with paths to the new baseline model weights.

    Training runs are long-running. Use TaskCreate (subagent) for each training run per CLAUDE.md workflow preferences.
  </action>
  <verify>
    <automated>aquapose --project ~/aquapose/projects/YH data status</automated>
  </verify>
  <done>
    - Baseline OBB model trained with mosaic=0.3 and registered with "baseline" tag
    - Baseline pose model trained with mosaic=0.1, imgsz=128, rect=True and registered with "baseline" tag
    - Both models appear in `data status` output
    - Project config.yaml updated with new model weight paths
  </done>
</task>
