  <action>
    Create `src/aquapose/training/data_cli.py` with:

    1. `@click.group("data") def data_group()` — "Manage training data stores."

    2. `@data_group.command("import")` with options:
       - `--config` (required, Path exists) — project config for path resolution
       - `--store` (required, Choice["obb", "pose"]) — which store to import into
       - `--source` (required, Choice["manual", "corrected", "pseudo"]) — data source type
       - `--input-dir` (required, Path exists) — YOLO-format directory with images/ and labels/
       - `--augment` (flag) — generate elastic augmentation variants on import
       - `--augment-count` (int, default 4) — number of variants per sample when --augment is set
       - `--augment-angle-range` (float, default 15.0) — max deformation angle in degrees
       - `--batch-id` (str, optional) — shared import batch ID for cross-type tracking
       - `--metadata-json` (str, optional) — JSON string of additional metadata to attach to all imported samples

       Implementation:
       - Resolve project dir from config, construct store path: `{project_dir}/training_data/{store}/store.db`
       - Create SampleStore instance
       - Scan input-dir for image files (glob `images/**/*.jpg` + `images/**/*.png`)
       - For each image, find matching label by stem in labels/ directory (same relative sub-path, e.g., images/train/foo.jpg -> labels/train/foo.txt)
       - Wrap the entire import loop in a single DB transaction for performance (call conn.execute("BEGIN") before loop, conn.commit() after)
       - Call store.import_sample() for each pair
       - **Augmentation (--augment flag, pose store only):**
         - If --store is "obb": print info message "Augmentation only applies to pose (crop-space) data, skipping --augment for OBB store." and skip augmentation.
         - If --store is "pose": for each newly imported sample (action == "imported"):
           1. Load the image: `cv2.imread(str(store.images_dir / f"{sample_id}.jpg"))`
           2. Parse the YOLO label file to extract keypoints: read `store.labels_dir / f"{sample_id}.txt"`, parse the pose line format (`cls cx cy w h x1 y1 v1 x2 y2 v2 ...`), extract coords as (N,2) array and visible as (N,) bool array, compute crop_w and crop_h from image shape
           3. Compute lateral_pad from the image dimensions (use crop_h * 0.18 as default lateral ratio, matching the project convention)
           4. Call `generate_variants(image=img, coords=coords, visible=visible, crop_w=crop_w, crop_h=crop_h, lateral_pad=lateral_pad, angle_range=(5.0, augment_angle_range))`
           5. For each returned variant dict: write variant["image"] to a temp file (cv2.imwrite), write variant["pose_line"] to a temp label file, call `store.add_augmented(parent_id=sample_id, image_path=tmp_img, label_path=tmp_label, metadata={"variant_tag": variant["variant_tag"]})`
           6. Clean up temp files after add_augmented copies them into the store
       - Track cascade-deleted augmentation count from upserts. If > 0, print warning: "Warning: {N} augmented variants were cascade-deleted during upserts. Re-run with --augment to regenerate."
       - Print summary: N imported, N upserted, N skipped, N augmented variants created

    3. `@data_group.command("convert")` with options:
       - `--coco-file` (required, Path exists) — path to COCO keypoints JSON
       - `--images-dir` (required, Path exists) — directory containing source images
       - `--output-dir` (required, Path) — output directory for YOLO-format data
       - `--type` (required, Choice["obb", "pose", "both"]) — what to generate
       - `--n-keypoints` (int, default 6) — number of keypoints for pose conversion
       - `--crop-width` (int, default 128) — crop width for pose conversion
       - `--crop-height` (int, default 64) — crop height for pose conversion
       - `--lateral-ratio` (float, default 0.18) — OBB lateral padding as fraction of median arc length
       - `--min-visible` (int, default 4) — minimum visible keypoints to include a fish (pose)
       - `--val-split` (float, default 0.2) — fraction of data for validation
       - `--seed` (int, default 42) — random seed for reproducible split
       - `--edge-threshold-factor` (float, default 2.0) — multiplier for edge keypoint extrapolation

       Implementation:
       - This command replaces scripts/build_yolo_training_data.py. It reuses the existing functions directly:
         - Import from scripts module: `from aquapose.training.geometry import extrapolate_edge_keypoints, format_obb_annotation, format_pose_annotation, pca_obb`
         - The functions `load_coco`, `compute_median_arc_length`, `generate_obb_dataset`, `generate_pose_dataset` currently live in scripts/build_yolo_training_data.py. **Move them** to a new location `src/aquapose/training/coco_convert.py` (pure data conversion functions, no CLI/argparse). Then import from there.
       - For --type "obb": call load_coco, compute_median_arc_length, generate_obb_dataset with all parameters
       - For --type "pose": call load_coco, compute_median_arc_length, generate_pose_dataset with all parameters
       - For --type "both": call both, print combined summary
       - Output is standard YOLO directory structure ready for `aquapose data import`
       - Print summary: N train / N val images per type

    4. Wire into main CLI: In `src/aquapose/cli.py`, add `from aquapose.training.data_cli import data_group` and `cli.add_command(data_group)`

    5. After convert command is working and tested: delete `scripts/build_yolo_training_data.py`

    **Important — moving functions from script to module:**
    The convert command must NOT reimplement the COCO→YOLO conversion logic. The functions in `scripts/build_yolo_training_data.py` (load_coco, parse_keypoints, compute_arc_length, compute_median_arc_length, affine_warp_crop, transform_keypoints, generate_obb_dataset, generate_pose_dataset) should be moved to `src/aquapose/training/coco_convert.py` as-is (they're already well-tested through usage). The CLI command is a thin Click wrapper around these existing functions. Do NOT include generate_seg_dataset or format_seg_annotation — seg training is not part of the current workflow.
