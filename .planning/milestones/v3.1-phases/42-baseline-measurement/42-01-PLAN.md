  <name>Task 1: Add outlier flagging to evaluation output module</name>
  <files>src/aquapose/evaluation/output.py</files>
  <action>
    Add an outlier flagging helper to `src/aquapose/evaluation/output.py` and a new `format_baseline_report` function that produces an outlier-annotated summary.

    1. Add a helper function `flag_outliers(values: dict[str, float], threshold_std: float = 2.0) -> set[str]`:
       - Compute mean and std of all values.
       - Return the set of keys whose value exceeds mean + threshold_std * std.
       - If fewer than 2 values or std is 0, return empty set (no outliers detectable).

    2. Add `format_baseline_report(tier1: Tier1Result, tier2: Tier2Result, fixture_name: str, frames_evaluated: int, frames_available: int) -> str`:
       - Start with the same content as `format_summary_table(...)` (call it internally or duplicate the formatting — prefer calling it and then post-processing the lines).
       - Apply outlier detection to Tier 1 per-camera mean_px values: compute outlier set via `flag_outliers({cam: vals["mean_px"] for cam, vals in tier1.per_camera.items()})`. For flagged cameras, append " *" to their row in the table.
       - Apply outlier detection to Tier 1 per-fish mean_px values similarly.
       - Apply outlier detection to Tier 2 per-fish dropout values: for each fish, collect non-None displacement values across dropout cameras, flag those exceeding 2 stdev. Append " *" to flagged entries.
       - Append a legend at the bottom: `"* = outlier (>2σ from mean)"`.

    The simplest correct approach: Call `format_summary_table` to get the base table. Then build a NEW report string that re-formats with outlier markers. This avoids fragile post-processing of the formatted string. The `format_baseline_report` function should produce its own formatted output using the raw Tier1Result/Tier2Result data directly, with asterisk annotations inline.

    Do NOT modify the existing `format_summary_table` function — it remains unchanged for Phase 41 compatibility.

    Create `scripts/measure_baseline.py` as a standalone Python script. This is the primary deliverable of Phase 42.

    1. **Argparse setup:**
       - Required positional argument: `fixture_path` (Path to the midline fixture NPZ file).
       - Optional `--n-frames` with default 15 (passed through to run_evaluation).
       - Help text: brief description of what the script does.

    2. **Main logic (in a `main()` function):**
       a. Parse args. Resolve fixture_path to an absolute Path.
       b. Validate fixture file exists (sys.exit with message if not).
       c. Call `run_evaluation(fixture_path, n_frames=args.n_frames)`.
       d. Generate the outlier-flagged report via `format_baseline_report(results.tier1, results.tier2, fixture_name=fixture_path.stem, frames_evaluated=results.frames_evaluated, frames_available=results.frames_available)`.
       e. Print the report to console.
       f. Save the report to `baseline_report.txt` next to the fixture file: `fixture_path.parent / "baseline_report.txt"`.
       g. Build baseline JSON by reading the eval_results.json that run_evaluation already wrote (at results.json_path), loading it, adding a `"baseline_metadata"` key with:
          - `"timestamp"`: ISO 8601 UTC string (datetime.utcnow().isoformat() + "Z")
          - `"fixture_path"`: str(fixture_path.resolve())
          - `"backend_identifier"`: `"aquapose.core.reconstruction.triangulation.triangulate_midlines"` (the module path of the current backend function)
       h. Write the augmented JSON to `fixture_path.parent / "baseline_results.json"` with indent=2.
       i. Print paths of saved files to console.

    3. **if __name__ == "__main__": main()** guard at bottom.

    4. **Imports needed:**
       - `argparse`, `json`, `sys` from stdlib
       - `pathlib.Path`, `datetime.datetime` from stdlib
       - `from aquapose.evaluation import run_evaluation`
       - `from aquapose.evaluation.output import format_baseline_report`

    The script is intentionally simple — a thin wrapper over Phase 41's harness with outlier annotation and metadata persistence.
