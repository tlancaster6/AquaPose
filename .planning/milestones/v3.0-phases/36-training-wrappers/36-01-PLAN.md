<objective>
Add `--mode seg` to `scripts/build_yolo_training_data.py` that converts COCO segmentation polygon annotations into per-detection-crop NDJSON datasets, matching the existing OBB and pose NDJSON patterns.

Purpose: DATA-01 requires a segmentation training data converter. The seg converter follows the same per-detection-crop architecture as the pose converter: for each OBB detection, warp the crop, then record all visible fish polygon masks in that crop. This produces the intermediate NDJSON format that the training wrapper (Plan 02) will convert to YOLO .txt labels.

Output: Updated `build_yolo_training_data.py` with `--mode` flag and `generate_seg_dataset` function; expanded test suite with `TestSegConverter` class.

**Add `format_seg_annotation()` function:**
```python
def format_seg_annotation(
    polygon: np.ndarray,  # (M, 2) in crop pixels
    crop_w: int,
    crop_h: int,
    class_id: int = 0,
) -> dict:
```
- Normalize polygon vertices to [0, 1] by dividing by (crop_w, crop_h)
- Clip to [0, 1]
- Return `{"class_id": class_id, "polygon": [[x1,y1], [x2,y2], ...]}`

**Add `generate_seg_dataset()` function** following the same per-detection-crop architecture as `generate_pose_dataset()`:

1. For each image in COCO, for each annotation that has keypoints:
   - Compute OBB from keypoints (same as pose: `extrapolate_edge_keypoints` + `pca_obb`)
   - Extract affine crop (same as pose: `affine_warp_crop`)
   - For each annotation in the **same image** (ALL fish, not just the target):
     - Get its COCO `segmentation` polygon(s)
     - If multi-ring (multiple polygons in `ann["segmentation"]`): keep largest ring by vertex count (`max(ann["segmentation"], key=len)`)
     - Parse the flat COCO polygon `[x1,y1,x2,y2,...]` into (M, 2) array
     - Transform polygon vertices through the same affine matrix using homogeneous coordinates (same math as `transform_keypoints` but without visibility masking — use direct affine multiplication)
     - Skip annotations where fewer than 3 polygon vertices fall within crop bounds `[0, crop_w) x [0, crop_h)`
     - Clip partial polygons to crop bounds
     - Call `format_seg_annotation()` for each valid polygon
   - Save crop image and record NDJSON line

2. The crop is identified by the **target** annotation (the one whose OBB defines the crop), but the NDJSON record's `annotations` list includes ALL visible fish polygons in that crop.

3. Output structure mirrors pose:
   - `output_dir/seg/images/{train,val}/` — crop images
   - `output_dir/seg/{train,val}.ndjson` — one JSON line per crop
   - `output_dir/seg/data.yaml` — standard YAML with `nc: 1`, `names: {0: fish}`

4. NDJSON record schema:
```json
{"image": "images/train/foo_001.jpg", "width": 128, "height": 64, "annotations": [{"class_id": 0, "polygon": [[x1,y1],[x2,y2],...]}]}
```

5. `data.yaml` content:
```yaml
path: .
train: train.ndjson
val: val.ndjson
nc: 1
names:
  0: fish
```

**Important details:**
- Polygon affine transform: convert flat COCO polygon `[x1,y1,x2,y2,...]` to (M,2) array, then apply `affine_matrix @ [x,y,1].T` for each vertex. This is the same math as `transform_keypoints` but operates on arbitrary-length polygon arrays.
- After affine transform, normalize by dividing x by crop_w, y by crop_h, then clip to [0, 1].
- The `--mode seg` path requires COCO annotations to have `segmentation` fields. If an annotation lacks `segmentation`, skip it (don't error).
- For seg mode, the `--annotations` COCO JSON should have segmentation polygons. The script should also accept annotations that have BOTH keypoints and segmentation (the OBB is derived from keypoints, polygons come from segmentation).
