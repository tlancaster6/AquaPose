  <name>Task 1: Add store methods for dataset assembly and persistence</name>
  <files>src/aquapose/training/store.py, tests/unit/training/test_store.py</files>
  <behavior>
    - test_save_dataset_stores_manifest: Save dataset with name, query recipe, and sample IDs. Retrieve by name.
    - test_save_dataset_overwrites_existing: Save dataset with same name twice. Second call replaces first.
    - test_get_dataset_returns_none_for_missing: Query nonexistent dataset returns None.
    - test_list_datasets_returns_all: Save two datasets, list returns both.
    - test_assemble_creates_symlinks: Call assemble method, verify symlinks created in YOLO directory structure pointing to store files.
    - test_assemble_creates_val_split: Assemble with val_fraction, verify train/val split is deterministic (same seed = same split).
    - test_symlinks_are_relative: Verify created symlinks use relative targets, not absolute.
  </behavior>
  <action>
    Add methods to SampleStore:

    1. `save_dataset(self, name: str, query_recipe: dict, sample_ids: list[str], split_seed: int) -> None`:
       - INSERT OR REPLACE into datasets table
       - Store query_recipe and sample_ids as JSON strings

    2. `get_dataset(self, name: str) -> dict | None`:
       - SELECT from datasets by name, parse JSON fields

    3. `list_datasets(self) -> list[dict]`:
       - SELECT all from datasets, parse JSON fields

    4. `assemble(self, name: str, query: dict, val_fraction: float = 0.2, seed: int = 42) -> Path`:
       - Query samples using self.query(**query)
       - Split into train/val using seeded random shuffle
       - Create directory: `self.root / "datasets" / name` with YOLO structure (images/train, images/val, labels/train, labels/val)
       - Create relative symlinks from dataset dir to store images/labels
       - Use `os.path.relpath(target, link_parent)` for portable relative symlinks
       - Write `dataset.yaml` with YOLO metadata (path, train, val, names, nc)
       - Call save_dataset() to persist manifest
       - Return dataset directory path

    5. `summary(self) -> dict`:
       - Returns counts by source, augmented count, excluded count, total, and dataset count
  </action>
  <verify>
    <automated>hatch run test -- tests/unit/training/test_store.py -x</automated>
  </verify>
  <done>SampleStore has dataset assembly and persistence methods with symlink creation and val splitting</done>
</task>

    1. `@data_group.command("assemble")` with options:
       - `--config` (required, Path exists)
       - `--store` (required, Choice["obb", "pose"])
       - `--name` (required, str) — dataset name (becomes subdirectory name)
       - `--source` (optional, str, multiple=True) — filter by source type(s)
       - `--tags-include` (optional, str, multiple=True)
       - `--tags-exclude` (optional, str, multiple=True)
       - `--val-fraction` (float, default 0.2)

    2. `@data_group.command("list")` with options:
       - `--config` (required, Path exists)
       - `--store` (required, Choice["obb", "pose"])
       - `--verbose` (flag) — show individual samples instead of summary
       Implementation: Open store, call store.summary(), print formatted table. If --verbose, list all samples with ID, source, tags, created_at.

    3. `@data_group.command("exclude")` with options:
       - `--config` (required, Path exists)
       - `--store` (required, Choice["obb", "pose"])
       - `--ids` (optional, str, multiple=True) — specific sample IDs
       - `--source` (optional, str) — exclude all samples from this source
       Implementation: Open store, resolve sample IDs from --ids or by querying --source, call store.exclude(), print count.

    4. `@data_group.command("remove")` with options:
       - `--config` (required, Path exists)
       - `--store` (required, Choice["obb", "pose"])
       - `--ids` (optional, str, multiple=True)
       - `--source` (optional, str)
       - `--purge` (flag, required for safety) — must pass to confirm deletion
       Implementation: Open store, resolve IDs, call store.remove() if --purge, print count. Without --purge, print error telling user to pass --purge to confirm.
