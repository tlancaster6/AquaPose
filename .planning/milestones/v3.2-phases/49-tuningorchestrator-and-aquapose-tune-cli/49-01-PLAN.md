**Create `src/aquapose/evaluation/tuning.py`:**

Module docstring: `"""TuningOrchestrator — grid sweep engine for association and reconstruction parameter tuning."""`

**Class: TuningOrchestrator**

Constructor `__init__(self, config_path: Path)`:
- Store `config_path` and `run_dir = config_path.parent`
- Use inline import of `load_config` (avoid top-level engine coupling, matching EvalRunner pattern)
- Load config via `load_config(config_path)`
- Load upstream caches: discover and load per-stage pickle caches from `run_dir / "diagnostics" / f"{stage}_cache.pkl"` using `load_stage_cache()`

**Method: `sweep_association(self, n_frames: int = 30, n_frames_validate: int = 100, top_n: int = 3) -> TuningResult`**

Sweep strategy (per CONTEXT.md locked decisions):
1. **Joint 2D grid** over `ray_distance_threshold x score_min` from `ASSOCIATION_DEFAULT_GRID`
2. For each combo in the joint grid:
   - `copy.copy(tracking_ctx)` for isolation (shallow copy per Phase 46 decision)
   - Patch `config.association` via `dataclasses.replace()` with combo params. **Cast `early_k` to int.**
   - Build `AssociationStage(patched_config).run(ctx_copy)` to get new `tracklet_groups`
   - Use the EXISTING `annotated_detections` from the midline cache + new `tracklet_groups` to build MidlineSets (reuse `_build_midline_sets()` helper, matching EvalRunner pattern)
   - Build `ReconstructionStage(...)` with baseline config and `.run(ctx_copy)` to get `midlines_3d`
   - Call `evaluate_association(midline_sets, n_animals)` and `evaluate_reconstruction(frame_results, fish_available)` — no tier2_result for fast sweep
   - Score: `(-assoc_metrics.fish_yield_ratio, recon_metrics.mean_reprojection_error)`
   - Print progress line: combo params, yield %, mean error
3. **Sequential carry-forward** for remaining params (`eviction_reproj_threshold`, `leiden_resolution`, `early_k`):
   - For each remaining param, sweep its grid values while keeping the winner from the joint grid (and any previously resolved params) fixed
   - Same score function
4. Collect all results, sort by score, take top-N
5. **Top-N validation** at `n_frames_validate`:
   - Re-run each top-N candidate at `n_frames_validate` with full Tier 1 + Tier 2 evaluation
   - For Tier 2: use `evaluate_reconstruction(frame_results, fish_available, tier2_result=compute_tier2(...))`
   - Note: if Tier 2 computation is complex, pass `tier2_result=None` and document as a known simplification
6. Return `TuningResult`

**Method: `sweep_reconstruction(self, n_frames: int = 30, n_frames_validate: int = 100, top_n: int = 3) -> TuningResult`**

Sweep strategy:
1. **1D sequential carry-forward**: `outlier_threshold` first, then `n_points` (which maps to `n_sample_points` in config)
2. For each param, for each value in grid:
   - `copy.copy(midline_ctx)` for isolation
   - Build `ReconstructionStage(calibration_path=..., backend=..., ..., outlier_threshold=val)` where `outlier_threshold` is passed as a keyword arg (forwarded via `**backend_kwargs`)
   - For `n_points` grid values: pass as `n_sample_points` when constructing the stage (actually, check if `n_sample_points` is a constructor param or if it's just in the config — it appears NOT to be a ReconstructionStage constructor param; the grid may need to be re-mapped or this param may only matter to the midline stage). **Investigate carefully**: `n_points` in the reconstruction grid likely maps to the backend's sample count. If it's not a stage constructor param, skip it or pass via `**backend_kwargs`.
   - `.run(ctx_copy)` to get `midlines_3d`
   - `evaluate_reconstruction(frame_results, fish_available)` — no tier2 for fast sweep
   - Score: `(recon_metrics.mean_reprojection_error, -recon_metrics.fish_reconstructed / max(recon_metrics.fish_available, 1))`
3. Top-N validation at `n_frames_validate` (same pattern as association)
4. Return `TuningResult`

**Dataclass: `TuningResult`** (frozen)
- `stage: str` — "association" or "reconstruction"
- `winner_params: dict[str, float]` — the winning parameter combination
- `baseline_metrics: dict[str, float]` — key metrics from baseline run
- `winner_metrics: dict[str, float]` — key metrics from winner
- `all_combos: list[dict]` — all sweep results for reporting
- `joint_grid_results: list[dict] | None` — for 2D yield matrix (association only)

**Helper functions** (module-level, prefixed with `_`):
- `_build_midline_sets(ctx, tracklet_groups, sampled_indices)` — adapted from EvalRunner._build_midline_sets(), takes explicit tracklet_groups param so we can pass the new ones from a sweep combo while using the midline cache's annotated_detections
- `_compute_association_score(assoc_metrics, recon_metrics) -> tuple[float, float]` — `(-assoc_metrics.fish_yield_ratio, recon_metrics.mean_reprojection_error)`
- `_compute_reconstruction_score(recon_metrics) -> tuple[float, float]` — `(recon_metrics.mean_reprojection_error, -recon_metrics.fish_reconstructed / max(recon_metrics.fish_available, 1))`

**Output formatting functions** (public, for CLI to call):
- `format_comparison_table(baseline_metrics: dict, winner_metrics: dict) -> str` — produces the before/after table with deltas. Metrics: yield %, mean error, max error, singleton rate, tier2 stability.
- `format_yield_matrix(joint_results: list[dict], param_a: str, values_a: list[float], param_b: str, values_b: list[float]) -> str` — produces the 2D yield % matrix for association sweeps
- `format_config_diff(stage_name: str, winner_params: dict, baseline_config) -> str` — produces a YAML snippet of changed keys. Use `yaml.dump()`. Convert float values that are whole numbers to int for cleaner output.

**CRITICAL IMPLEMENTATION NOTES:**
- Use inline imports for engine modules (matching EvalRunner pattern): `from aquapose.engine.config import load_config` inside methods, NOT at module level
- Use `copy.copy()` (NOT `copy.deepcopy()`) for context isolation — shallow copy is safe per Phase 46 decision
- Cast `early_k` grid values to `int` before `dataclasses.replace()` on AssociationConfig
- Map `n_points` grid key to `n_sample_points` config field for reconstruction
- Print progress lines to stdout as each combo completes (per CONTEXT.md: "Progress lines always")
- The `n_frames` param controls frame sampling via `select_frames()` from `evaluation/metrics.py`
- For association sweeps, the upstream cache is `tracking_cache.pkl`. For reconstruction sweeps, the upstream cache is `reconstruction_cache.pkl` (which contains midline data) — actually use `midline_cache.pkl` since it has `tracklet_groups` + `annotated_detections`.
- For association sweeps, after running AssociationStage, also need to run ReconstructionStage to get `midlines_3d` for reconstruction metrics. The midline cache's `annotated_detections` is reused (MidlineStage NOT re-run).
