  <files>src/aquapose/core/inference.py, tests/unit/core/test_inference.py, src/aquapose/core/__init__.py</files>
  <behavior>
    - Test 1: predict_with_oom_retry with no OOM calls predict_fn on all inputs in correct chunk sizes
    - Test 2: predict_with_oom_retry with batch_size=0 sends all inputs in one call
    - Test 3: When predict_fn raises torch.cuda.OutOfMemoryError, batch halves and retry succeeds
    - Test 4: When predict_fn raises RuntimeError with "CUDA out of memory", batch halves and retry succeeds
    - Test 5: When RuntimeError does NOT contain "CUDA out of memory", it propagates immediately
    - Test 6: When batch_size=1 fails with OOM, the exception propagates (no infinite loop)
    - Test 7: BatchState.effective_batch_size persists across calls -- second call uses reduced size
    - Test 8: BatchState.oom_occurred is True after any OOM event
    - Test 9: Empty inputs list returns empty list without calling predict_fn
  </behavior>
  <action>
Create `src/aquapose/core/inference.py` with:

1. `BatchState` dataclass (mutable, NOT frozen):
   - `effective_batch_size: int | None = None`
   - `original_batch_size: int | None = None`
   - `oom_occurred: bool = False`

2. `predict_with_oom_retry(predict_fn, inputs, max_batch_size, state)` function:
   - `predict_fn: Callable[[list], list]` -- takes a chunk, returns results
   - `inputs: list` -- full input list
   - `max_batch_size: int` -- 0 means no limit (batch everything)
   - `state: BatchState` -- mutable state persisted across calls
   - Logic: use `state.effective_batch_size` if set, else `max_batch_size`, else `len(inputs)`
   - Set `state.original_batch_size` on first call
   - Loop: chunk inputs by effective batch size, call predict_fn per chunk, extend results
   - On `torch.cuda.OutOfMemoryError` or `RuntimeError` containing "CUDA out of memory":
     - If effective <= 1: raise (don't loop forever)
     - Call `torch.cuda.empty_cache()`
     - Halve effective, set `state.effective_batch_size`, set `state.oom_occurred = True`
     - `logger.warning("CUDA OOM during inference: halving batch size to %d", effective)`
     - Retry from scratch (not mid-chunk)
   - On other RuntimeError: re-raise immediately

For tests, mock `torch.cuda.OutOfMemoryError` and `torch.cuda.empty_cache` since tests run without GPU. Use a fake predict_fn that raises OOM on first call then succeeds.

Add `BatchState` and `predict_with_oom_retry` to `src/aquapose/core/__init__.py` exports.
