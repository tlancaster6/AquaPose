Refactor `src/aquapose/training/pseudo_label_cli.py` to support both consensus and gap label generation:

**CLI changes:**
- Add `--consensus` flag (is_flag=True, default=False) -- generates Source A labels
- Add `--gaps` flag (is_flag=True, default=False) -- generates Source B gap-fill labels
- Add `--min-cameras` flag (type=int, default=3, show_default=True) -- contributing-camera floor for gap detection
- Add validation: if neither --consensus nor --gaps is passed, raise `click.ClickException("At least one of --consensus or --gaps is required.")`
- Update function signature: `def generate(config_path, lateral_pad, max_camera_residual, consensus, gaps, min_cameras)`
- `--max-camera-residual` applies only to consensus labels (per user decision)
- `--lateral-pad` applies to both consensus and gap labels

**Directory restructure:**
- Consensus output: `pseudo_labels/consensus/{obb,pose}/{images,labels}/train/`
- Gap output: `pseudo_labels/gap/{obb,pose}/{images,labels}/train/`
- Each subset gets its own `dataset.yaml` (inside `{subset}/{obb,pose}/dataset.yaml`)
- Each subset gets its own `confidence.json` (inside `{subset}/confidence.json`)

**Gap-specific additions (only when --gaps is passed):**
- Load InverseLUT via dynamic import: `importlib.import_module("aquapose.calibration.luts")` to get `load_inverse_luts`. Construct `_LutConfigFromDict` from the frozen config's `lut` section. Call `load_inverse_luts(pipeline_config.calibration_path, lut_config)`.
- Note: `_LutConfigFromDict` likely already exists in this file from Phase 62 or can be found in the codebase. Check first; if not present, define it locally with a comment referencing Phase 62 pattern.
- Fail fast if `context.detections` is None or `context.tracks_2d` is None when --gaps is passed (these are needed for classification)
- Pre-build a frame-to-tracklet index per camera for O(1) lookup (Pitfall 3 from RESEARCH.md): `frame_tracklet_index: dict[str, dict[int, list[Tracklet2D]]]` -- for each camera, map frame_idx to list of tracklets active at that frame. Build once before the main loop.

**Main loop refactoring:**
- When both --consensus and --gaps are set, process both for the same frame in one pass (share frame reads, Pitfall 6)
- For each frame_idx with fish in midlines_3d:
  - **Consensus path** (if --consensus): Same as current logic -- iterate fish, iterate `per_camera_residuals` keys, call `generate_fish_labels()`, collect to consensus output dirs
  - **Gap path** (if --gaps): For each fish, call `detect_gaps(midline, inverse_lut, proj_models, context.detections[frame_idx], frame_tracklet_index_for_frame, min_cameras)`. For each (cam_id, reason), call `generate_gap_fish_labels(midline, proj_models[cam_id], img_w, img_h, keypoint_t_values, lateral_pad)`. Collect to gap output dirs. Add `gap_reason` and `n_source_cameras` to confidence sidecar entries.
  - `frame_tracklet_index_for_frame` should be derived from the pre-built index: for each camera, get the list of tracklets active at this frame_idx
- Read frame images once if any labels (consensus or gap) exist for that frame
- Write images to appropriate subset directories (consensus and/or gap)

**Gap confidence sidecar format:**
```python
{
    "image_name": {
        "labels": [
            {
                "fish_id": int,
                "confidence": float,
                "raw_metrics": dict,
                "gap_reason": str,  # "no-detection" | "no-tracklet" | "failed-midline"
                "n_source_cameras": int,
            }
        ]
    }
}
```
