<objective>
Add dataset assembly (symlink-based, with manifest and pseudo-excluded val split), cross-store status, listing, soft-delete, include (reverse exclude), and hard-delete CLI commands to complete the data management workflow.

Purpose: Assembly produces training-ready YOLO directories without file duplication. Status provides at-a-glance navigability across both stores. List/exclude/include/remove provide full lifecycle management.
Output: Six new CLI commands wired into the `data` group, store methods for dataset persistence.

Note: The existing `dataset_assembly.py` is kept as-is for now. It handles pseudo-label-specific assembly logic (confidence filtering, gap-reason filtering, run-directory scanning) that the store-based assembly does not yet cover. Once the store has been used for a full retrain cycle, evaluate whether to deprecate dataset_assembly.py or merge its specialized features into the store workflow.
</objective>

    - test_assemble_creates_deterministic_split: Assemble twice with same seed, verify identical train/val split.

    - test_summary_returns_counts: Import mixed samples, call summary, verify source breakdown and totals.

  <action>
    Add methods to SampleStore:

    1. `save_dataset(self, name: str, query_recipe: dict, sample_ids: list[str], split_seed: int) -> None`:
       - INSERT OR REPLACE into datasets table
       - Store query_recipe and sample_ids as JSON strings

    2. `get_dataset(self, name: str) -> dict | None`:
       - SELECT from datasets by name, parse JSON fields

    3. `list_datasets(self) -> list[dict]`:
       - SELECT all from datasets, parse JSON fields

    4. `assemble(self, name: str, query: dict, val_fraction: float = 0.2, seed: int = 42, pseudo_in_val: bool = False) -> Path`:
       - Query samples using self.query(**query)
       - **Val split logic:**
         - If pseudo_in_val=False (default): partition samples into val-eligible (source in ["manual", "corrected"]) and train-only (source == "pseudo")
         - Draw val set from val-eligible samples only (seeded random shuffle, take first N)
         - Remaining val-eligible samples go to train
         - All pseudo samples go to train
         - If pseudo_in_val=True: all samples are val-eligible (simple random split as before)
       - Create directory: `self.root / "datasets" / name` with YOLO structure (images/train, images/val, labels/train, labels/val)
       - If directory already exists, remove it first (clean reassembly)
       - Create relative symlinks from dataset dir to store images/labels
       - Use `os.path.relpath(target, link_parent)` for portable relative symlinks
       - Write `dataset.yaml` with YOLO metadata (path, train, val, names, nc)
       - Call save_dataset() to persist manifest
       - Return dataset directory path

    5. `summary(self) -> dict`:
       - Returns dict with keys: total, by_source (dict), augmented_count, excluded_count, dataset_count, model_count
  </action>
