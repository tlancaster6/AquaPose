    **EVAL-04: Per-keypoint reprojection error computation.**

    Add a new function `compute_per_point_error()` in reconstruction.py:
    ```python
    def compute_per_point_error(
        frame_results: list[tuple[int, dict[int, Midline3D]]],
        projection_models: dict[str, Any],  # camera_id -> RefractiveProjectionModel
        n_body_points: int = 15,
    ) -> dict[int, dict[str, float]] | None:
    ```
    - For each Midline3D in frame_results:
      - Evaluate BSpline at `np.linspace(0, 1, n_body_points)` to get 3D sample points (N, 3)
      - For each camera in `m3d.per_camera_residuals.keys()` (the cameras that observed this fish):
        - Get projection model for that camera
        - Project 3D points: `proj_px, valid = model.project(torch.from_numpy(pts_3d.astype(np.float32)))`
        - CRITICAL: `proj_np = proj_px.detach().cpu().numpy()` (CUDA safety)
        - Skip invalid projections
        - Need corresponding 2D midline points to compute error. Since we don't have per-camera 2D points in Midline3D, we must receive them as a parameter OR compute error differently.

    **Key insight:** The `Midline3D.per_camera_residuals` dict stores mean residual per camera, but NOT per-point residuals. We need the 2D observed midline points. These come from MidlineSets (fish_id -> camera_id -> Midline2D). So `compute_per_point_error` must also accept `midline_sets: list[MidlineSet]` to get the observed 2D points.

    Revised signature:
    ```python
    def compute_per_point_error(
        frame_results: list[tuple[int, dict[int, Midline3D]]],
        midline_sets_by_frame: dict[int, MidlineSet],  # frame_idx -> MidlineSet
        projection_models: dict[str, Any],  # camera_id -> projection model
        n_body_points: int = 15,
    ) -> dict[int, dict[str, float]] | None:
    ```
    Returns `{point_index: {"mean_px": float, "p90_px": float}}` or None if no data.

    For each (frame_idx, midline_dict) in frame_results:
      For each (fish_id, m3d) in midline_dict:
        Evaluate spline at n_body_points -> pts_3d (N, 3)
        Get MidlineSet for this frame: midline_sets_by_frame[frame_idx][fish_id]
        For each (camera_id, midline_2d) in the fish's camera map:
          If camera_id in projection_models:
            Project pts_3d through model -> proj_2d (N, 2)
            obs_2d = midline_2d.points  # (N, 2) -- same N sample points
            For each point j: error[j] = norm(proj_2d[j] - obs_2d[j]) if valid
            Accumulate errors per point index

    After all frames: per point index, compute mean and p90 from accumulated errors.

    **EVAL-05: Curvature-stratified quality.**

    Add a new function `compute_curvature_stratified()` in reconstruction.py:
    ```python
    def compute_curvature_stratified(
        frame_results: list[tuple[int, dict[int, Midline3D]]],
        midline_sets_by_frame: dict[int, MidlineSet],
    ) -> dict[str, dict[str, float]] | None:
    ```
    Returns `{"Q1": {"mean_error_px": float, "p90_error_px": float, "count": int, "curvature_range": str}, ...}` or None.

    For each (frame_idx, fish_id, m3d):
      Get MidlineSet for this frame/fish
      Find camera with highest mean point_confidence:
        `best_cam_midline = max(cam_map.values(), key=lambda m: np.mean(m.point_confidence) if m.point_confidence is not None else 1.0)`
      Compute curvature: `compute_curvature(best_cam_midline.points)` -- per user decision, use 2D midline points
      Guard: if `best_cam_midline.points.shape[0] < 3`, skip (per pitfall 5)
      Store (curvature, m3d.mean_residual) pair

    After all frames:
      If fewer than 4 pairs, return None
      Compute quartile bin edges: `np.quantile(curvatures, [0.25, 0.5, 0.75])`
      Bin using `np.digitize(curvatures, bin_edges)` -> indices 0-3
      Per bin: compute mean_error_px, p90_error_px, count

    Import `compute_curvature` from `aquapose.training.pseudo_labels`.

    **ReconstructionMetrics extension:**

The EvalRunner will call them separately and inject the results. Use `dataclasses.replace()` to add the results to the metrics object, OR extend `evaluate_reconstruction` signature with optional params.

    This keeps the evaluator pure -- it just passes through pre-computed values.
