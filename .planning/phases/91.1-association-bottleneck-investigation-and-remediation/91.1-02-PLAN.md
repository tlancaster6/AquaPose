**Vectorize `_compute_frame_residual_centroid`:**

The current code calls `cast_ray` with a single pixel for each "other" tracklet in a loop. Restructure to batch all others for this frame:

1. Cast the target ray once (single pixel) -- keep as-is since it's 1 call
2. Collect all other centroids that have this frame into a single array
3. Group by camera_id (each camera needs its own `cast_ray` call since LUTs are per-camera)
4. For each unique other camera, batch all its pixels into one `cast_ray` call, then `ray_ray_closest_point_batch` against the (tiled) target ray
5. Collect all distances and return mean

Since there are typically only 1-3 "others" per frame (few cameras share a frame), the main win here is eliminating per-call overhead. The structure should be:
```python
# Cast target ray (1 pixel)
t_origins, t_dirs = lut_target.cast_ray(pix_t)
t_o = t_origins[0].cpu().numpy().astype(np.float64)
t_d = t_dirs[0].cpu().numpy().astype(np.float64)

distances: list[float] = []
for other, fm in zip(others, other_frame_maps, strict=True):
    if frame not in fm:
        continue
    idx_other = fm[frame]
    centroid_other = other.centroids[idx_other]
    lut_other = forward_luts[other.camera_id]
    pix_o = torch.tensor([[float(centroid_other[0]), float(centroid_other[1])]], dtype=torch.float32)
    o_origins, o_dirs = lut_other.cast_ray(pix_o)
    o_o = o_origins[0].cpu().numpy().astype(np.float64)
    o_d = o_dirs[0].cpu().numpy().astype(np.float64)
    dist = ray_ray_closest_point_batch(
        t_o.reshape(1, 3), t_d.reshape(1, 3),
        o_o.reshape(1, 3), o_d.reshape(1, 3),
    )[0]
    distances.append(float(dist))
```
This is a minor change since the centroid path has few iterations, but it replaces the scalar function consistently.

Note: The centroid path has very few iterations per frame (one per other camera), so the main benefit is consistency. The keypoint path is the real bottleneck.
