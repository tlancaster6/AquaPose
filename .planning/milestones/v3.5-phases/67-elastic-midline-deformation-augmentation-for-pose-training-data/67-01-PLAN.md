  <behavior>
    ## Keypoint deformation

    deform_keypoints_c_curve(coords, angle_deg) -> deformed_coords:
    - Input: (6, 2) keypoints in crop pixel space, signed angle in degrees (10-30 range)
    - Compute arc-length parameterization t in [0, 1] along the keypoint polyline
    - Displace each keypoint perpendicular to the local midline direction by: offset = R - R*cos(t * total_angle), where R = chord_length / (2 * sin(total_angle/2))
    - Along-axis displacement: R*sin(t * total_angle) - t * chord_length
    - Both head and tail shift symmetrically (centroid of deformed points matches centroid of original)
    - Test: 6 collinear keypoints at y=100, x=[50,70,90,110,130,150], angle=20 deg -> endpoints shift symmetrically, midpoint has max lateral displacement
    - Test: angle=0 -> no displacement (identity)
    - Test: negative angle -> mirror of positive angle

    deform_keypoints_s_curve(coords, amplitude_deg) -> deformed_coords:
    - Input: (6, 2) keypoints, signed amplitude in degrees
    - Displace perpendicular to local midline by: offset = A * sin(pi * t) where t is arc-length fraction [0, 1] and A is derived from amplitude_deg and body length
    - Re-center so centroid matches original
    - Test: collinear keypoints, positive amplitude -> middle keypoints displaced one way, first/last near zero
    - Test: negative amplitude -> mirror displacement

    ## TPS image warp

    tps_warp_image(image, src_points, dst_points, crop_w, crop_h) -> warped_image:
    - Uses 6 keypoint control points + 4 corner identity anchors (10 total)
    - Corner anchors: (0,0), (W-1,0), (W-1,H-1), (0,H-1) mapped to themselves
    - Uses cv2.BORDER_REPLICATE for out-of-bounds fill
    - Implementation: build TPS via OpenCV ThinPlateSplineShapeTransformer or scipy RBF
    - Test: identity mapping (src == dst) -> output matches input
    - Test: non-identity mapping -> output differs from input, shape preserved

    ## Label generation

    generate_deformed_labels(deformed_coords, visible, crop_w, crop_h, lateral_pad) -> dict:
    - Calls pca_obb on deformed keypoints to get new OBB
    - Calls format_obb_annotation and format_pose_annotation
    - Returns {"obb_line": list[float], "pose_line": list[float], "obb_corners": np.ndarray}
    - Test: deformed keypoints produce valid normalized label values in [0, 1]

    ## Variant generation (high-level)

    generate_variants(image, coords, visible, crop_w, crop_h, lateral_pad, angle_range, n_c, n_s) -> list[dict]:
    - Generates n_c C-curve variants (symmetric +/- angles) and n_s S-curve variants (symmetric +/- amplitudes)
    - Default: 2 C-curves (opposite signs) + 2 S-curves (opposite signs) = 4 variants
    - Each dict: {"image": np.ndarray, "coords": np.ndarray, "visible": np.ndarray, "obb_line": ..., "pose_line": ..., "variant_tag": str}
    - Test: 4 variants returned with correct tags ("c_pos", "c_neg", "s_pos", "s_neg")

  <implementation>
    Create src/aquapose/training/elastic_deform.py with:

    1. `deform_keypoints_c_curve(coords, angle_deg)` - Uniform arc displacement.
       - Compute chord length from first to last keypoint
       - Arc-length parameterize t in [0, 1] for each keypoint
       - Compute perpendicular direction at each point from local tangent
       - Apply circular arc formula for lateral + longitudinal displacement
       - Re-center to preserve centroid

    2. `deform_keypoints_s_curve(coords, amplitude_deg)` - Sinusoidal displacement.
       - Convert amplitude_deg to pixel displacement: A = chord_length * tan(amplitude_deg * pi / 180)
       - Displace perpendicular by A * sin(pi * t)
       - Re-center to preserve centroid

    3. `tps_warp_image(image, src_points, dst_points, crop_w, crop_h)` - TPS warp.
       - Use cv2.ThinPlateSplineShapeTransformer (Claude's discretion per CONTEXT.md)
       - Concatenate 6 keypoints + 4 corner anchors for both src and dst
       - Apply backward mapping to produce warped image
       - borderMode=cv2.BORDER_REPLICATE

    4. `generate_deformed_labels(deformed_coords, visible, crop_w, crop_h, lateral_pad)` - Label formatting.
       - Call pca_obb(deformed_coords, visible, lateral_pad) for new OBB
       - Compute normalized bbox from OBB for pose label
       - Call format_obb_annotation and format_pose_annotation
       - Return dict with both label lines

    5. `generate_variants(image, coords, visible, crop_w, crop_h, lateral_pad, angle_range=(10, 30))` - High-level API.
       - Sample one angle from angle_range (midpoint or random per Claude's discretion)
       - Generate 4 variants: C+, C-, S+, S-
       - For each: deform keypoints, TPS warp image, generate labels
       - Return list of variant dicts

    Note on fish body length: ~90-150px in crop space. Calibrate warp magnitudes so 10-30 degree bends produce only a few pixels of endpoint displacement. This means actual pixel displacements will be small (roughly 5-20px for the most displaced keypoint).

    Update src/aquapose/training/__init__.py to export the new public functions and add to __all__.
