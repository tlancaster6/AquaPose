<objective>
Implement the DltBackend — a stripped-down reconstruction backend that uses confidence-weighted DLT triangulation with single-pass outlier rejection, B-spline fitting, and water surface rejection.

Purpose: Replaces the complex multi-strategy triangulation logic (pairwise exhaustive search for small camera sets, residual rejection for large sets, epipolar refinement, orientation alignment) with a single, uniform algorithm: triangulate all → reject outliers → re-triangulate inliers → fit spline.

Output: `backends/dlt.py` with `DltBackend` class, updated `backends/__init__.py` registry, comprehensive unit tests.
</objective>

       - `__init__(self, calibration_path, outlier_threshold=DEFAULT_OUTLIER_THRESHOLD, n_control_points=DEFAULT_N_CONTROL_POINTS, low_confidence_fraction=DEFAULT_LOW_CONFIDENCE_FRACTION)`:
         Loads calibration via the same pattern as TriangulationBackend._load_models (copy that static method or import it — the executor should replicate it to avoid coupling to old backend).
       - `reconstruct_frame(self, frame_idx: int, midline_set: MidlineSet) -> dict[int, Midline3D]`:
         For each fish_id in midline_set:
           a. Read `n_body_points = len(next(iter(cam_midlines.values())).points)`.
           b. For each body point i:
              - Gather pixel observations and confidence weights from all cameras (skip NaN points).
              - Skip cameras not in self._models.
              - Cast rays for all cameras via model.cast_ray().
              - **Ray-angle filter**: For 2-camera case, check angle between rays; skip if < 5 degrees. For >2 cameras, keep all (DLT handles it).
              - Determine weighting: if any midline has `point_confidence is not None` and not all 1.0, use `sqrt(confidence)` weights via `weighted_triangulate_rays`. Otherwise use `triangulate_rays` (unweighted).
              - **Triangulate** all cameras together (single strategy — no camera-count branching).
              - **Water surface rejection**: if point Z <= water_z, drop this body point.
              - **Compute per-camera reprojection residuals**: reproject 3D point into each camera, compute pixel distance to observed point.
              - **Outlier rejection**: cameras with residual > outlier_threshold are rejected.
              - If <2 inlier cameras remain, drop the body point.
              - **Re-triangulate** with inlier cameras only (same weighting logic).
              - **Water surface rejection** again on re-triangulated point.
              - Record: 3D point, inlier cam IDs, mean residual, per-camera residuals.
           c. If fewer than `min_body_points` valid body points, skip this fish.
           d. Build arc-length parameter u = [i / (n_body_points - 1) for valid indices].
           e. Call `fit_spline(u_param, pts_3d, knots, min_body_points)` from utils.
           f. Convert half-widths to world metres via `pixel_half_width_to_metres` from utils. Use same interpolation logic as existing backend (interp1d for full body profile).
           g. Compute spline-based per-camera residuals (reproject fitted spline, compare to observed 2D).
           h. Flag `is_low_confidence` when fraction of body points with <3 inlier cameras > `low_confidence_fraction`.
           i. Build and return `Midline3D`.

    **Key differences from existing TriangulationBackend:**

    - NO camera-count branching (no 2-cam / 3-7 cam / 8+ cam paths)
