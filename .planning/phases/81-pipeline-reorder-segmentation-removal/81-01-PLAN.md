  Rename the class from `MidlineStage` to `PoseStage`. Key changes to `run()`:
  - PoseStage reads `context.detections` (no longer reads `context.tracklet_groups` to decide which detections to process — it processes ALL detections per the "pose-on-all-detections" decision).

    ```python

    ```
  - Do NOT call `_keypoints_to_midline` — the 6→15 upsampling moves to ReconstructionStage (Plan 02).
  - Do NOT create `AnnotatedDetection` wrapper objects.
  - Do NOT write to `context.annotated_detections`.
  - Do NOT call orientation resolution.
  - The stage still needs `frame_source` to read video frames for cropping. It still uses `extract_affine_crop` and `invert_affine_points` from `crop.py`.
  - Update the `PoseEstimationBackend` to have a method (or adapt existing `process_batch`/`process_single`) that returns raw keypoints + confidences instead of `AnnotatedDetection` objects. The backend should return `(kpts_xy_crop, kpts_conf, visible_mask)` tuples, and PoseStage handles the `invert_affine_points` back-projection to full-frame and writes onto Detection objects.
  - Keep `BatchState` and `predict_with_oom_retry` usage for batch inference.
  - Update `__all__` to `["PoseStage"]`.
  - Update class docstring: "Stage 2: Runs pose estimation on all detections, enriches Detection objects with raw anatomical keypoints."
