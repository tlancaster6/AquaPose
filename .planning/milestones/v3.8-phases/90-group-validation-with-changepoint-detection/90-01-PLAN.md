  <action>
Create `src/aquapose/core/association/validation.py` with:

1. **`ValidationConfigLike` Protocol** (follows `RefinementConfigLike` pattern from refinement.py):
   - `eviction_reproj_threshold: float` — reused as changepoint threshold
   - `min_cameras_validate: int` — minimum cameras to run validation (default will be 2 in config)
   - `validation_enabled: bool` — toggle
   - `min_segment_length: int` — minimum frames per split segment (default 10)
   - `keypoint_confidence_floor: float` — below this, keypoints excluded from residual computation

2. **`validate_groups(groups, forward_luts, config) -> list[TrackletGroup]`** — public API, mirrors `refine_clusters()` signature. For each group with enough cameras:
   - Compute per-tracklet residual series via `_compute_tracklet_residuals()`
   - Apply decision tree via `_classify_tracklet()`: keep (>50% consistent) / split (changepoint found) / evict (uniformly bad)
   - Split tracklets via `_split_tracklet_at()` with unique new track_ids
   - After splits/evictions, check thin group dissolution (1 camera remaining -> dissolve to singletons)
   - Recompute `per_frame_confidence` and `consensus_centroids` for cleaned group (copy `_compute_frame_consensus` and `_compute_per_frame_confidence` from refinement.py — do NOT import)
   - Assign unique fish_ids to new singleton groups

3. **`_compute_tracklet_residuals(target, others, forward_luts, config)`** — for each frame in target, cast rays from target's confident keypoints and from each other tracklet's matching keypoints, compute mean ray-ray distance. Skip frames with fewer than 2 cameras having confident keypoints. If target has `keypoints=None`, fall back to centroid-only ray distance (port `_compute_tracklet_distances` pattern from refinement.py). Return `(valid_frames: list[int], residuals: list[float])`.

4. **`_find_changepoint_by_run(residuals, frames, threshold, min_segment_length) -> int | None`** — classify each frame as consistent (residual < threshold) or inconsistent. Find the longest contiguous consistent run. If the consistent segment is >= min_segment_length AND remaining frames outside it are >= min_segment_length, return the end index of the longest consistent run (split point). Otherwise return None.

5. **`_classify_tracklet(residuals, frames, threshold, min_segment_length) -> tuple[str, int | None]`** — decision tree:
   - If >50% frames are consistent: return ("keep", None)
   - Else try `_find_changepoint_by_run()`: if found, return ("split", split_idx)
   - Else: return ("evict", None)

6. **`_split_tracklet_at(tracklet, split_idx, id_before, id_after) -> tuple[Tracklet2D, Tracklet2D]`** — construct two new Tracklet2D from sliced arrays. Guard `keypoints`/`keypoint_conf` with `if is not None` checks. Use tuple slicing for tuple fields, numpy slicing for ndarray fields.

CRITICAL per user decision:
- Residuals are per-tracklet vs rest of group (exclude target from consensus)
- All 6 keypoints contribute equally (confidence floor from Phase 88 already filters)
- Frames with fewer than 2 cameras having confident keypoints are skipped
- Changepoint threshold reuses `eviction_reproj_threshold` — one parameter
- Inconsistent segments become singleton candidates (Phase 91 handles recovery)

Copy `_compute_frame_consensus` and `_compute_per_frame_confidence` from refinement.py into validation.py (do NOT import from refinement — it will be deleted in Plan 02). These centroid-based helpers are unchanged — consensus_centroids must remain centroid-based for downstream compatibility.

Log split/eviction counts per group at INFO level for debugging.

Create `tests/unit/core/association/test_validation.py` following the test behaviors listed above. Use MockForwardLUT and MockValidationConfig patterns from test_refinement.py. For the swap test, create a MockForwardLUT that returns different rays based on pixel coordinates — the "swapped" tracklet should have centroids that match cam_a's geometry for the first half and diverge for the second half.
