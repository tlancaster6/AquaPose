    **2. Create `src/aquapose/training/common.py`:**
    Shared training utilities used by all subcommands. Must include:

    - `EarlyStopping` class: tracks best metric value, returns True when patience exceeded. Constructor takes `patience: int`, `mode: str` ("min" or "max"). Method `step(metric: float) -> bool` returns True if should stop. Property `best: float` returns best metric seen.

    - `MetricsLogger` class: writes metrics to both console and a CSV file. Constructor takes `output_dir: Path`, `fields: list[str]`. Method `log(epoch: int, **metrics: float) -> None` prints one-line epoch summary and appends to `metrics.csv`. Format: `Epoch {epoch}/{total} - field1: {val:.4f} - field2: {val:.4f} - time: {elapsed:.1f}s`. The CSV has a header row and one row per epoch.

    - `save_best_and_last(model: nn.Module, output_dir: Path, metric: float, best_metric: float, metric_name: str) -> tuple[Path, float]`: Saves `last_model.pth` always, saves `best_model.pth` when metric improves (comparison direction based on metric_name convention: "loss" = lower is better, anything else = higher is better). Returns (best_model_path, updated_best_metric).

    - `make_loader(dataset, batch_size: int, shuffle: bool, device: str, num_workers: int = 4) -> DataLoader`: Creates DataLoader with pin_memory=(device startswith "cuda"), persistent_workers=(num_workers > 0).

    **3. Create `src/aquapose/training/datasets.py`:**

This is a fresh rewrite using the old code as behavioral reference (per user decision). The public API and behavior must match — same constructor signatures, same output shapes, same augmentation pipeline. Keep `UNET_INPUT_SIZE = 128` constant.
