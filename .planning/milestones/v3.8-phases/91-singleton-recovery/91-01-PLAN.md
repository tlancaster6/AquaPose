  <action>
Create `src/aquapose/core/association/recovery.py` with:

1. **RecoveryConfigLike Protocol** (runtime_checkable):
   - `recovery_enabled: bool`
   - `recovery_residual_threshold: float`
   - `recovery_min_shared_frames: int`
   - `recovery_min_segment_length: int`
   - `keypoint_confidence_floor: float` (reused from scoring/validation)

2. **`recover_singletons(groups, forward_luts, config)` entry point**:
   - Partition groups into multi-tracklet groups and singletons (len(tracklets) == 1)
   - If no singletons or recovery disabled, return early
   - Score all singletons against all multi-tracklet groups
   - Greedy best-first whole assignment pass (sort by residual ascending, assign best first, enforce same-camera overlap before scoring)
   - For remaining unassigned singletons, attempt split-assign sweep
   - Reassemble final group list with updated groups and remaining singletons
   - Assign unique fish_ids to remaining singletons (max_id + 1, +2, ...)

3. **`_score_singleton_against_group(singleton, group, forward_luts, config)` -> float | None**:
   - Find shared frames between singleton and group's frame union
   - Guard: return None if shared < recovery_min_shared_frames
   - For each shared frame, triangulate group's per-keypoint 3D positions on-demand using ray-ray midpoints from group member tracklets
   - Cast singleton's rays for confident keypoints, compute _point_to_ray_distance for each keypoint against group's triangulated position
   - Return arithmetic mean over all (frame, keypoint) pairs with valid data
   - Fallback: if singleton.keypoints is None, use centroid-only ray-to-consensus distance

4. **`_triangulate_group_keypoints_for_frame(frame, group_tracklets, forward_luts, kpt_conf_floor, n_keypoints)` -> np.ndarray | None**:
   - For each keypoint k, collect rays from group tracklets that have confident keypoint k in this frame
   - Compute pairwise ray-ray midpoints, average to get consensus_3d[k]
   - Returns (K, 3) array or None if < 2 cameras have any confident keypoint

5. **`_has_camera_overlap(singleton, group, detected_only=True)` -> bool**:
   - Check if singleton's camera_id matches any group tracklet's camera_id with overlapping detected frames
   - Only frame_status == "detected" frames count as overlap

6. **`_attempt_split_assign(singleton, groups, forward_luts, config, next_track_id)` -> result or None**:
   - Guard: skip if singleton.frames length < 2 * recovery_min_segment_length
   - For each valid split point, slice singleton into two segments
   - Score each segment against all groups (re-check same-camera overlap per segment)
   - Pick split that minimizes total residual where both segments match DIFFERENT groups below threshold
   - Return split info or None

7. **`_point_to_ray_distance(point, origin, direction)` -> float**: Copy from refinement.py (standalone, no cross-module import).

8. **`_slice_tracklet(tracklet, start, end, new_id)` -> Tracklet2D**: Slice all fields. Copy pattern from validation.py `_split_tracklet_at`.

Use `.cpu().numpy()` for all LUT tensor conversions. Use `torch.tensor(..., dtype=torch.float32)` for cast_ray input.

Create `tests/unit/core/association/test_recovery.py` with tests matching the behavior list above. Use MockForwardLUT pattern from test_validation.py (mock cast_ray returning predictable origins/directions). Test the full recover_singletons() entry point and individual helpers.
