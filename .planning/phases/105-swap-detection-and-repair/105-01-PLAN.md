Create `src/aquapose/core/reid/swap_detector.py` with:

**Dataclasses:**
- `ReidEvent(frame, fish_a, fish_b, cosine_margin, detection_mode, action)` — provenance record
- `SwapDetectorConfig(cosine_margin_threshold=0.15, window_frames=10, proximity_threshold_m=0.15, scan_frame_stride=1, scan_dense_window=20, scan_gap_stride=10, min_window_frames=3)` — all thresholds configurable

**Pure functions (testable without GPU/data):**
- `_confirm_swap(emb_a_pre, emb_a_post, emb_b_pre, emb_b_post, threshold) -> tuple[bool, float]` — cross-pattern check. Computes cross_a = dot(a_post, b_pre), cross_b = dot(b_post, a_pre), self_a = dot(a_post, a_pre), self_b = dot(b_post, b_pre). margin = min(cross_a - self_a, cross_b - self_b). Returns (margin > threshold, margin).
- `_compute_mean_embedding(embeddings: np.ndarray) -> np.ndarray | None` — mean then L2-normalize. Returns None if input is empty.
- `_find_proximity_events(frame_index, fish_id, points, threshold_m) -> list[tuple[int, int, int]]` — pairwise 3D centroid distance per frame, Z_WEIGHT applied. Returns deduplicated events (collapse consecutive frames of same pair into single event at the midpoint frame).

**SwapDetector class:**
- `__init__(self, run_dir: Path, config: SwapDetectorConfig | None = None, projection_head_path: Path | None = None)` — loads H5 data, optionally loads embeddings.npz for reuse, optionally loads projection head for embedding transformation
- `run(self, mode: Literal["seeded", "scan"]) -> list[ReidEvent]` — dispatches to seeded or scan mode
- `_run_seeded(self) -> list[ReidEvent]` — reads `/midlines/swap_events`, for each event: gathers embeddings in pre/post window (window_frames), computes mean embeddings per fish, calls `_confirm_swap`. Events with fewer than `min_window_frames` frames of embedding data are skipped (action="skipped_insufficient_data").
- `_run_scan(self) -> list[ReidEvent]` — (1) finds proximity events via `_find_proximity_events`, (2) embeds densely around each (scan_dense_window), (3) scans remaining gaps at scan_gap_stride using a sliding window, (4) applies cross-pattern check to each candidate
- `_get_embeddings(self, fish_id: int, frame_start: int, frame_end: int) -> np.ndarray | None` — looks up in cached embeddings.npz first, then computes on-the-fly from video if needed. Returns (M, 768) array or None if no valid crops.
- `repair(self, events: list[ReidEvent]) -> Path` — copies midlines_stitched.h5 to midlines_reid.h5, applies fish_id swaps for events with action="repaired" (same propagation logic as `apply_swap_repairs`), writes `/reid_events/` group, returns output path. NOTE: repair only relabels fish_id; 3D points are mathematically identical — document this invariance in a docstring.

**H5 provenance writing:**
- `_write_reid_events(h5_path, events)` — writes `/reid_events/` group with separate datasets per field (frame, fish_a, fish_b, cosine_margin, detection_mode, action) using `h5py.string_dtype()` for string fields. Follows the pattern from RESEARCH.md.

**On-the-fly embedding:**
- For on-the-fly crop extraction, reuse the crop extraction pattern from `EmbedRunner.run()` — load chunk caches, match tracklet detections to stitched fish_ids using the prestitch-to-stitched ID map. The `SwapDetector` should instantiate `FishEmbedder` only when on-the-fly embedding is needed (lazy init).
- If `projection_head_path` is provided, load a `torch.nn.Linear` (or `nn.Sequential`) projection head and apply it to 768-dim embeddings before comparison. Use `.cpu().numpy()` per CLAUDE.md pitfall.

**Important implementation details:**
- Use `from aquapose.core.stitching import Z_WEIGHT, SwapEvent` for imports
- Read H5 swap events via `f["midlines"]["swap_events"]` (NOT `f["/swap_events/"]`) per pitfall #1
- After averaging embeddings, re-normalize to unit length (mean of unit vectors is not unit)
- Apply Z_WEIGHT to z-component when computing centroid distances per pitfall #7
- Handle empty embedding windows gracefully (skip with warning) per pitfall #2
- Apply confirmed events in chronological order per pitfall #4
