- **Clean path (augment=False):** derive visibility from COCO `v > 0` AND coordinates in-bounds (`0 <= x < 128, 0 <= y < 128`). Convert to normalized [0,1]. Zero coordinates for invisible keypoints.
- **Augmented path (augment=True):** wrap image as `tv_tensors.Image` (uint8, before float conversion), wrap keypoints as `tv_tensors.KeyPoints(kp_pixel, canvas_size=(128, 128))`. - Return `(image_tensor, kp_flat, visibility_mask)` where visibility_mask is `torch.Tensor` of shape `(n_keypoints,)` bool.

CRITICAL coordinate protocol (from RESEARCH.md):
1. Load image, resize to 128x128
2. Map COCO coords to 128px space: `x_px = float(x) / orig_w * 128`
3. Apply transforms in pixel space (tv_tensors.KeyPoints needs pixel coords matching canvas_size)
4. Detect OOB after transform
5. Normalize to [0,1]: `x_norm = x_px / 128`
6. Zero coordinates for invisible keypoints

CRITICAL: For the augmented path, the image must be uint8 when passed to transforms (ColorJitter works on uint8; convert to float32 AFTER transforms). For the clean path, keep existing behavior (numpy -> permute -> float / 255).

**5. Add `_masked_mse_loss` function** (after `_mean_keypoint_error`):
```python
def _masked_mse_loss(
    pred: torch.Tensor,
    target: torch.Tensor,
    visibility: torch.Tensor,
) -> torch.Tensor:
    """MSE loss computed only on visible keypoints.

    Args:
        pred: (B, n_keypoints * 2) predicted coordinates in [0, 1].
        target: (B, n_keypoints * 2) ground-truth coordinates in [0, 1].
        visibility: (B, n_keypoints) bool mask, True = visible.

    Returns:
        Scalar loss tensor.
    """
    vis_exp = visibility.float().repeat_interleave(2, dim=1)  # (B, n_kp*2)
    diff_sq = (pred - target) ** 2
    n_visible = vis_exp.sum().clamp(min=1.0)
    return (diff_sq * vis_exp).sum() / n_visible
```
