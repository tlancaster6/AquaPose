  Function signature:
  ```python
  def generate_fixture(
      yaml_config_path: Path | str,
      association_overrides: dict[str, object] | None = None,
      output_path: Path | str | None = None,
  ) -> Path:
  ```

  Implementation:
  1. Import inside the function (to avoid circular imports and heavy top-level loads):
     - `from aquapose.engine.config import load_config, AssociationConfig`
     - `from aquapose.engine.pipeline import PosePipeline`
     - `from aquapose.engine.diagnostic_observer import DiagnosticObserver`
     - `from aquapose.engine.stages import build_stages`
     - `import dataclasses`
     - `import tempfile`

  2. Load the pipeline config from YAML: `config = load_config(yaml_config_path)`

  3. If `association_overrides` is provided, patch the association config:
     ```python
     patched_assoc = dataclasses.replace(config.association, **association_overrides)
     config = dataclasses.replace(config, association=patched_assoc)
     ```

  4. If `output_path` is None, use a temp directory: `output_path = Path(tempfile.mkdtemp()) / "midline_fixtures.npz"`

  5. Create a DiagnosticObserver with `output_dir` set to `output_path.parent` and `calibration_path` from config.

  6. Build stages via `build_stages(config)`, create `PosePipeline(stages, config, observers=[observer])`, run it.

  7. The observer auto-exports the fixture on pipeline complete. Return the fixture path.

  Docstring should note:
  - This runs the FULL pipeline (detection + tracking + association + midline) — requires GPU, video files, and calibration data
  - Each call produces a complete fixture with the given association config
  - Use for parameter sweeps where each config needs a fresh fixture

  Note: Do NOT add tests for this function — it requires full pipeline infrastructure (GPU, video, calibration). It will be verified manually via `tune_association.py` in Plan 02.
