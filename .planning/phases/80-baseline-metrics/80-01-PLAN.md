  <name>Task 2: Build measure_baseline_tracking.py script</name>
  <files>scripts/measure_baseline_tracking.py</files>
  <action>
Create `scripts/measure_baseline_tracking.py` modeled on `scripts/investigate_occlusion.py`. The script should:

**Arguments (argparse):**
- `--config` — path to project config.yaml (required)
- `--camera` — camera ID, default `e3v83eb`
- `--start-frame` — default 3300
- `--end-frame` — default 4500
- `--n-animals` — expected number of fish, default 9
- `--output-dir` — where to save video and metrics JSON, default `./baseline_tracking_output`

**Execution flow:**
1. Load config using the same `_load_config()` pattern from `investigate_occlusion.py` to resolve OBB model weights and video directory
2. Initialize `ultralytics.YOLO` with the OBB weights
3. Initialize `OcSortTracker(camera_id=camera, min_hits=1, det_thresh=0.1)` — use `min_hits=1` per research recommendation for honest baseline measurement
4. Open video with `cv2.VideoCapture`, seek to `start_frame`
5. Loop from `start_frame` to `end_frame`:
   a. Read frame
   b. Run OBB detection (conf=0.1, same as investigate_occlusion.py)
   c. Feed detections to `tracker.update(fidx, det_objects)` — use ABSOLUTE frame index (fidx = current global frame number), not loop-relative index
   d. Store per-frame detection+track info for video annotation
6. Call `tracker.get_tracklets()` to get `list[Tracklet2D]`
7. Call `evaluate_tracking(tracklets)` to get `TrackingMetrics`
8. Call `evaluate_fragmentation_2d(tracklets, n_animals)` to get `FragmentationMetrics`
9. Print metrics summary to console in a readable format
10. Save metrics as JSON to `output_dir/baseline_metrics.json`

**Video annotation (second pass or during first pass):**
- Re-read frames from start_frame to end_frame
- For each frame, draw OBB outlines colored by track_id (use a color palette, cycle if >10 colors)
- Overlay track_id text near each OBB center in matching color
- Gray outlines for untracked detections (if any)
- Write annotated video to `output_dir/baseline_tracking.mp4` using `cv2.VideoWriter` with mp4v codec

**Detection → tracker bridge:**
Follow the exact pattern from `investigate_occlusion.py` for converting YOLO OBB results to the format `OcSortTracker.update()` expects. Read `investigate_occlusion.py` carefully to understand the `DetectionObject` or equivalent structure.

**CUDA pitfall:** Use `.cpu().numpy()` when converting any tensor to numpy (CLAUDE.md requirement).

**Console output format:**
```
=== OC-SORT Baseline Tracking Metrics ===
Camera: e3v83eb | Frames: 3300-4500 (1200 frames)

Track count:        {N}
Length min:          {N} frames
Length max:          {N} frames
Length median:       {N} frames
Length std:          {N:.1f} frames
Coast frequency:    {N:.3f}
Detection coverage: {N:.3f}

Fragmentation:
  Total gaps:       {N}
  Mean gap length:  {N:.1f}
  Continuity ratio: {N:.3f}
  Births:           {N}
  Deaths:           {N}

Target: 9 tracks, zero fragmentation
Gap to target: {analysis}
```
