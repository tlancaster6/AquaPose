<task type="auto">
  <name>Task 1: Write performance validation script</name>
  <files>scripts/perf_validate.py</files>
  <action>
Create `scripts/perf_validate.py` that does three things:

1. **parse_timing(path: Path) -> dict[str, float]**: Parse a TimingObserver `timing.txt` file. Each stage line matches `^\s+(\S+)\s+([\d.]+)s`. TOTAL line matches `^\s+TOTAL\s+([\d.]+)s`. Return dict mapping stage name to seconds (include "TOTAL" key).

2. **compare_eval(baseline_path: Path, post_path: Path) -> list[dict]**: Load two `eval_results.json` files and compare metrics with these tolerances:
   - detection: total_detections=0, mean_confidence=0.0
   - tracking: track_count=0, detection_coverage=0.0
   - association: fish_yield_ratio=0.02, singleton_rate=0.05
   - midline: total_midlines=0, mean_confidence=0.001
   - reconstruction: mean_reprojection_error=0.5, fish_reconstructed=2
   Return list of failure dicts with keys: stage, metric, baseline, post, tolerance, delta.

3. **generate_report(baseline_timing: dict, post_timing: dict, failures: list[dict] | None, baseline_run: str, post_run: str, baseline_eval_available: bool) -> str**: Generate markdown report with:
   - Header with date, baseline run ID, post-optimization run ID
   - Timing comparison table: Stage | Before (s) | After (s) | Speedup -- for all stages plus TOTAL row
   - Speedup = before/after formatted as "X.Xx"
   - Correctness section: "PASS" if failures is None or empty, "FAIL" with failure table otherwise
   - If baseline_eval_available is False, note that eval comparison was skipped due to stale caches

4. **CLI entry point** (argparse): Accept `--baseline-timing` (path to baseline timing.txt), `--post-run-dir` (path to post-optimization run directory), `--baseline-run-dir` (path to baseline run directory, optional), `--output` (path to write report markdown). The script:
   - Parses baseline timing from `--baseline-timing`
   - Parses post timing from `<post-run-dir>/timing.txt`
   - If `--baseline-run-dir` is provided and both `eval_results.json` files exist, runs compare_eval
   - Writes report to `--output`
   - Prints report to stdout

No external dependencies -- stdlib only (json, re, pathlib, argparse, datetime).
  </action>
