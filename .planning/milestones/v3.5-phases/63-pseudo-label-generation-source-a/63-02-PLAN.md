  <action>
    Create `src/aquapose/training/pseudo_label_cli.py` with:

    **Config dataclass** (NOT frozen, plain dataclass for defaults):
    ```python
    @dataclass
    class PseudoLabelConfig:
        lateral_pad: float = 40.0  # pixels, for OBB lateral padding
        max_camera_residual_px: float = 15.0  # per-camera residual threshold
        edge_factor: float = 2.0  # edge extrapolation factor
    ```

    **Click group and command:**
    ```python
    @click.group("pseudo-label")
    def pseudo_label_group(): ...

    @pseudo_label_group.command("generate")
    @click.option("--config", "-c", required=True, type=click.Path(exists=True), help="Path to frozen run config.yaml")
    @click.option("--lateral-pad", type=float, default=40.0, help="OBB lateral padding in pixels")
    @click.option("--max-camera-residual", type=float, default=15.0, help="Per-camera residual threshold in pixels")
    def generate(config, lateral_pad, max_camera_residual): ...
    ```

    **generate() implementation:**
    1. Load frozen config via `load_config(yaml_path=config)` to get PipelineConfig
    2. Fail fast if `config.midline.keypoint_t_values is None` with `click.ClickException`
    3. Resolve `run_dir` from config's `output_dir` (the frozen config is stored IN the run_dir, so `Path(config).parent` IS the run_dir)
    4. Load diagnostic caches via `_load_and_merge_chunks(run_dir)` from `aquapose.evaluation.runner`; fail fast if None
    5. Load calibration via `load_calibration_data(config.calibration_path)`
    6. Build projection models: `{cam_id: build_projection_model(cal.cameras[cam_id], cal) for cam_id in cal.ring_cameras}`
    7. Open VideoFrameSource for frame extraction (context manager)
    8. Create output dirs: `run_dir/pseudo_labels/{obb,pose}/{images,labels}/train/`
    9. Iterate frames: `for frame_idx, fish_dict in enumerate(context.midlines_3d):`
       - Read frame images: `frames = frame_source.read_frame(frame_idx)` -> `{cam_id: np.ndarray}`
       - For each `fish_id, midline` in `fish_dict.items()`:
         - For each `cam_id` in contributing cameras (from `midline.per_camera_residuals.keys()`):
           - Call `generate_fish_labels(midline, proj_models[cam_id], img_w, img_h, keypoint_t_values, lateral_pad, max_camera_residual, cam_id)`
           - If result is None, skip
           - Collect label lines per (frame_idx, cam_id) image
       - For each (frame_idx, cam_id) with labels:
         - Save image: `cv2.imwrite(images_dir / f"{frame_idx:06d}_{cam_id}.jpg", frames[cam_id])`
         - Write OBB label file: one line per fish
         - Write pose label file: one line per fish
         - Record confidence entries in sidecar dict
    10. Write `dataset.yaml` for OBB (path, train, nc=1, names={0: "fish"})
    11. Write `dataset.yaml` for pose (add kpt_shape and flip_idx)
    12. Write `confidence.json` sidecar with per-image confidence metadata
    13. Print summary: total images, total labels, mean confidence

    **Key design notes:**
    - No train/val split at this stage (Phase 65 handles that). All labels go to `train/` subdirectory.
    - Images are written as JPG (cv2.imwrite) -- not symlinks, since training may happen on different machines.
    - One label .txt file per image, containing one line per fish visible in that camera+frame.
    - The confidence sidecar is a single JSON file mapping `"{frame_idx:06d}_{cam_id}"` to `{"labels": [{"fish_id": int, "confidence": float, "raw_metrics": dict}, ...]}`.

    **Write tests** in `tests/unit/training/test_pseudo_label_cli.py`:
    - Create a minimal synthetic diagnostic cache (pickle with PipelineContext containing 2 frames, 1 fish each, with mock Midline3D objects that have per_camera_residuals for 2 cameras)
    - Create a minimal mock calibration JSON (or mock load_calibration_data)
    - Create a minimal frozen config YAML in a tmp directory
    - Test: `generate` command with CliRunner produces output directories with expected structure
    - Test: `generate` command fails with clear error when keypoint_t_values is None
    - Test: `generate` command fails with clear error when no diagnostic caches found
    - Mock RefractiveProjectionModel.project to return known pixel values (avoid needing real calibration)
    - Mock VideoFrameSource.read_frame to return synthetic images

    Note: Use monkeypatch/mock for projection model and frame source since we cannot run real refractive projection in unit tests without calibration data. The core math is tested in Plan 01's tests.
