  <action>
Create `scripts/investigate_occlusion.py` — a standalone argparse CLI script that:

**CLI arguments:**
- `--project-config` (required): Path to project config YAML (e.g., `~/aquapose/projects/YH/config.yaml`)
- `--camera` (required): Camera ID to analyze (e.g., `e3v831e`)
- `--start-frame` / `--end-frame`: Frame range (default: full video)
- `--crop-region`: Pixel region as `x1,y1,x2,y2` (default: `263,225,613,525` per CONTEXT.md)
- `--output-dir`: Where to write output video and stats (default: current dir)
- `--conf-threshold`: Detection confidence threshold (default: 0.2)
- `--conf-sweep`: Flag to run confidence sweep mode instead of single-threshold video

**Core pipeline (single-threshold mode):**
1. Load project config YAML to get `detection.weights_path`, `midline.weights_path`, `video_dir`, `calibration_path` (resolve relative paths against `project_dir`)
2. Open the single camera's video file directly with `cv2.VideoCapture` (do NOT use `VideoFrameSource` — we only need one camera, no undistortion needed for visual investigation of detector behavior in pixel space)
3. For each frame in range:
   a. Run OBB detection via `ultralytics.YOLO(obb_weights).predict(frame, conf=conf_threshold)` — extract Detection objects (bbox, confidence, angle, obb_points) from results
   b. Feed detections to `OcSortTracker.update()` for track ID assignment
   c. For each detection, extract affine crop using the same 3-point affine method from `PoseEstimationBackend._extract_crop()` (reuse the corner logic: pts[2]=TL, pts[1]=TR, pts[3]=LB, with side_h>side_w rotation handling)
   d. Run pose model via `ultralytics.YOLO(pose_weights).predict(crop)` on each crop
   e. Extract ALL instances from `kp.xy` (not just `[0]`) — iterate `range(len(kp.xy))` to get primary and secondary instances
   f. Transform keypoints back to frame coordinates using `cv2.invertAffineTransform(M)` then matrix multiply
4. After OC-SORT processes all frames, call `get_tracklets()` to get final track assignments
5. Re-iterate frames for rendering (or store per-frame data in first pass):
   - Crop the frame to the crop region
   - Draw OBB boxes: tracked detections colored by track ID (reuse `_PALETTE_BGR` from `evaluation/viz/overlay.py`), untracked detections with conf >= 0.5 in red, untracked with conf < 0.5 in gray
   - Draw keypoints with connections (nose->head->spine1->spine2->spine3->tail, indices 0->1->2->3->4->5) — circle radius proportional to per-keypoint confidence (e.g., `max(2, int(conf * 10))`)
   - Secondary pose instances: dashed connections (draw as dotted line segments), lighter/desaturated color
   - NO confidence text overlay per user decision
   - Write frame to `cv2.VideoWriter` (mp4v codec, 30fps)
6. Save per-frame statistics as JSON alongside the video:
   - Frame index, number of detections, number tracked, number of pose instances per detection, multi-instance flag, per-keypoint confidences

**Confidence sweep mode (`--conf-sweep`):**
1. Run OBB detection on all frames at conf=0.05 (below sweep range to capture everything)
2. For each threshold in `[0.10, 0.15, 0.20, 0.25, 0.30, 0.35, 0.40, 0.45, 0.50]`:
   - Filter detections by threshold
   - Count: total detections, mean/median/min/max detections per frame
3. Print results as a markdown table to stdout AND save to `confidence_sweep.md` in output dir

**Important implementation details:**
- All tensor-to-numpy via `.cpu().numpy()` (CUDA safety)
- Ultralytics OBB corners: `obb.xyxyxyxy` returns `[right-bottom, right-top, left-top, left-bottom]` — true TL is index 2
- The OC-SORT tracker needs Detection objects with bbox as (x,y,w,h). Build Detection from OBB results.
- For the two-pass approach (detection+tracking then rendering): store all per-frame data (detections, track assignments, keypoints) in a list-of-dicts during pass 1, render in pass 2. This is simpler than trying to match OC-SORT's delayed track assignment to individual frames.
- Actually, OcSortTracker.update() returns per-frame assignments — check its API. If it gives frame-level track IDs, single-pass is fine. If tracklets are only available after get_tracklets(), use two passes.
- Script should work without importing any aquapose modules that require the full pipeline to be initialized. Load YOLO models directly, build affine crops manually (copy the 3-point math, ~20 lines). This keeps the script truly standalone.
