    Extract a shared `_run_training()` function in `cli.py` that encapsulates the common orchestration pattern currently copy-pasted across the `yolo_obb`, `seg`, and `pose` CLI commands.

    1. Create `_run_training()` with this signature:
       ```python
       def _run_training(
           ctx: click.Context,
           model_type: str,
           data_dir: str | None,
           tag: str | None,
           cli_args: dict,
           *,
           train_kwargs: dict | None = None,
       ) -> None:
       ```

    2. The function body performs (in order):
       a. Resolve `config_path` and `project_dir` from `ctx`
       b. Default `data_dir` to `project_dir / "training_data" / model_type` if None
       c. `create_run_dir(config_path, model_type)`
       d. `setup_file_logging(run_dir, f"train-{model_type}")`
       e. Update `cli_args["data_dir"]` with the resolved value
       f. `snapshot_config(run_dir, cli_args, dataset_dir=Path(data_dir))`
       g. Import and call `train_yolo` from `.yolo_training` with `model_type` and forwarded kwargs. Build kwargs from `cli_args` (excluding "data_dir", "tag") merged with `train_kwargs` if provided.
       h. `write_summary(run_dir, results_csv, ...)`
       i. `try/except` around `register_trained_model(...)` — on failure, print yellow warning (matching existing obb/pose pattern). This is what fixes the seg bug.
       j. `print_next_steps(run_dir, model_type, best_path)`

    3. Refactor each CLI command:
       - `yolo_obb`: Build `cli_args` dict from options, call `_run_training(ctx, "obb", data_dir, tag, cli_args)`. ~10 lines of option assembly + one call.

       - `pose`: Same pattern with `"pose"`. Pass `rect` in `cli_args`. ~10 lines.
