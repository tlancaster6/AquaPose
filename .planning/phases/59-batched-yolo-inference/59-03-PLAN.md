<objective>
Add `process_batch()` to both midline backends and refactor `MidlineStage.run()` to collect all crops per frame and call `process_batch()` once via the OOM retry utility.

Purpose: Eliminate per-crop serial `model.predict()` calls (potentially 60-100+ per frame across 12 cameras x N fish), replacing them with a single batched GPU forward pass. This addresses ~13% of pipeline wall time (midline stage).
Output: Batched midline inference with OOM resilience, backward-compatible `process_frame()` preserved.
</objective>

```python
def process_batch(
    self,

    metadata: list[tuple[Detection, str, int]],  # (det, cam_id, frame_idx)

```

Implementation:
1. If `self._model is None` or `not crops`: return null AnnotatedDetection for each metadata entry.
2. Extract crop images: `crop_images = [c.image for c in crops]`
3. Call `self._model.predict(crop_images, conf=self._conf, verbose=False, batch=len(crop_images))`
4. For each `(result, crop, (det, cam_id, frame_idx))` in zip:
   - Call `self._extract_mask([result], crop.image.shape)` -- NOTE: `_extract_mask` expects a list of Results and takes `results[0]`. In batch mode, each `result` is a single Results object. Wrap it in a list: `self._extract_mask([result], crop.image.shape)`.

   - Else: call `self._skeletonize_and_project(mask_np, crop.M, cam_id, frame_idx)`
   - Build AnnotatedDetection with the result
5. Return list in positional correspondence.

Refactor `MidlineStage.__init__()`:
- Accept optional `midline_batch_crops: int = 0` parameter.
- Create `self._batch_state = BatchState()`.
- Store `self._batch_size = midline_batch_crops`.

Refactor `MidlineStage.run()` -- replace the `self._backend.process_frame(...)` call with a batched path:

For each frame in the main loop:
1. **Collect phase (CPU):** Iterate cameras and detections, extract crops via `self._backend._extract_crop(det, frame)` (both backends have identical `_extract_crop`). Build parallel lists:
   - `crops: list[AffineCrop]` -- all crops for this frame
   - `metadata: list[tuple[Detection, str, int]]` -- `(det, cam_id, frame_idx)` per crop
   - `cam_det_indices: list[tuple[str, int]]` -- `(cam_id, det_index_within_camera)` for redistribution
   - Handle crop extraction failures: if `_extract_crop` raises, append a null entry to a separate list and skip adding to crops/metadata.

2. **Predict phase (GPU):** Call `predict_with_oom_retry(self._backend.process_batch_predict, crops_and_metadata, self._batch_size, self._batch_state)`.

   ACTUALLY -- the OOM retry utility expects `predict_fn(chunk) -> list` where chunk is a list of inputs. The inputs here are `(crop, metadata_tuple)` pairs. Define a local predict function:
   ```python
   def _batch_predict(items):
       crops_chunk = [item[0] for item in items]
       meta_chunk = [item[1] for item in items]
       return self._backend.process_batch(crops_chunk, meta_chunk)
   ```
   Then call: `batch_results = predict_with_oom_retry(_batch_predict, list(zip(crops, metadata)), self._batch_size, self._batch_state)`

3. **Redistribute phase:** Map `batch_results` back to `dict[str, list[AnnotatedDetection]]` using `cam_det_indices`. Initialize `annotated: dict[str, list[AnnotatedDetection]] = {cam_id: [] for cam_id in camera_ids}`. For each `(result, (cam_id, _det_idx))` in zip(batch_results, cam_det_indices), append result to `annotated[cam_id]`. Also append null entries for failed crops in the correct positions.

4. Append annotated to `annotated_per_frame`.

After the main loop, if `self._batch_state.oom_occurred`, log recommendation:
`logger.info("Midline batch size was reduced to %d due to CUDA OOM. Consider setting midline.midline_batch_crops=%d in config.", ...)`

Update `build_stages()` in `engine/pipeline.py` to pass `midline_batch_crops=config.midline.midline_batch_crops` to `MidlineStage`.

Update existing midline stage tests to verify the batched path works. Add a test confirming process_batch is called once per frame instead of process_frame being called (when using batched mode).

IMPORTANT: The filtering logic (tracklet_groups, group_det_index, _filter_frame_detections) must still be applied BEFORE crop collection. The batched path collects crops only for filtered_dets, not all detections.

IMPORTANT: The orientation resolution (`_apply_orientation`) still runs AFTER the batched loop, operating on the annotated_per_frame output -- this is unchanged.
